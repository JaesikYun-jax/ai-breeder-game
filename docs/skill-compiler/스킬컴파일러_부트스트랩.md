## 1. 작품 개요

- **로그라인**: 대각성 후에도 유일하게 F급으로 남은 SI 개발자가, 스킬을 수정 가능한 코드로 읽는 오리지널 상태창을 자각하고, 약한 스킬의 코드를 편집하여 사기 콤보를 조합하며 먼치킨으로 성장하다가, 시스템 자체가 인류의 의념을 수확하는 농장임을 발견한다.
- **장르 태그**: [현대 판타지], [헌터물], [먼치킨], [시스템 해부], [F급 성장], [코딩 메타]
- **타겟 플랫폼**: 문피아 (1차) → 카카오페이지/네이버시리즈 (2차 확장)
- **예상 분량**: 전체 200~300화 (3부) / 1부 80화 / 1회당 5,000~7,000자 (공백 포함)
- **시점**: 1인칭 (안강해)
- **톤 & 무드**: 자조/냉소, 사이다, IT 블랙코미디, 시스템 스릴러, 직장인 공감
- **톤 공식**: 사이다 4 : 자조 4 : 미스터리 2

## 2. 세계관 설정

### 2-1. 시간선

| 시점 | 사건 | 서사 기능 |
|---|---|---|
| 현재 - 10년 | 강해(당시 18세)에게만 상태창이 보임. 정신과 입원, 사회적 고립 | 고립의 원인. "미친놈" 낙인 |
| 현재 - 5년 | 첫 게이트 출현. 몬스터 약소, 각성자 극소수 | 세계 변화의 전조 |
| 현재 - 1년 | **대각성(Great Awakening)**: 전 인류 동시 상태창 개방. 게이트 폭증 | 강해의 "진짜 미친 건 아니었네" 자각, 그러나 "왜 나만 다른 버전이지" |
| **현재 (EP001)** | 헌터 직업 정착. 협회/길드/아카데미 체계 구축 완료. 강해는 대각성 후에도 각성 판정 불가 = F급 유일 사례 | 서사 시작점. 두 세계 모두에서 최하층 |

### 2-2. 전문 분야: 한국 SI/SM 업계 백엔드 개발

**안강해의 직업적 위치**:
- 중소 SI 2차 협력사 4년차 백엔드 개발자
- 원청 대기업 빌딩에 상주 파견. 원청 직원과 같은 공간, 다른 대우
- 연봉 약 4,000만 원대. 미드레벨 진입기 (기본기 있으나 아키텍처 결정권 없음)
- 기술 스택: Java/Spring, MySQL, AWS 기초, Git

**IT-헌터 구조 동형성** (R3 반영):

| IT 업계 | 헌터 사회 | 서사적 겹침 |
|---|---|---|
| 원청 → 1차 → 2차 → 프리랜서 | S → A → B → ... → F | 하위에서 시작, 계층 돌파 |
| 단가 깎기, 야근 강제 | 위험 던전 하청, 드롭 아이템 상납 | 위험은 하위가, 이익은 상위가 |
| "협력사분" 호칭 | "F급" 호칭 | 보이지 않는 벽 |
| 기술 탈취 (성과 원청 귀속) | 코드 편집 기법 길드 독점 시도 | "내 능력인데 네 자산이라고?" |
| PM의 비현실적 약속 | 길드 마스터의 불가능한 던전 공략 약속 | 위에서 지른 불, 아래서 끄기 |

### 2-3. 전문 분야 깊이 전략

**원칙**: IT/프로그래밍 용어는 이 작품에서 단순 비유가 아니라 **능력 시각화 자체**이므로, 일반적인 "전문 분야 비유"와 구분한다.

| 구분 | 용도 | EP당 빈도 | 예시 |
|---|---|---|---|
| **코드 블록** (능력 시각화) | 스킬 코드를 직접 보여주는 장면 | 1~2회 | `if (target.armor < 0.3) { apply_bleed(3s); }` |
| **인라인 코드** (능력 내러티브) | 1인칭 서술 속 코드 언급 | 자유 | "`damage = 0.8 * STR`이라는 한 줄이 보였다" |
| **IT 비유** (일상 비유) | 능력 외적인 IT 용어 사용 | 0~1회 | "하도급은 세계를 넘어도 하도급이다" |

**코드 블록 톤 가이드**:
- 코드 자체를 화려하게 미화하지 않는다. 담담하게, 기능적으로
- 강해의 자조 한 줄로 마무리: "이런 게 D급이다", "다섯 줄짜리 파편"
- 큰 코드 블록(5~15줄)은 빌드 룸 신, 새 스킬 분석 신에 한정

**IT 개발 용어 12개 대응표** (R3 반영):

| 개발 용어 | 소설 내 대응 | 서사적 쾌감 |
|---|---|---|
| 디버깅 | 스킬 코드 오류 찾기 → 강화 | 탐정 쾌감 |
| 리팩토링 | 스킬 구조 재설계 (기능 동일, 효율 증가) | 최적화 쾌감 |
| 컴파일 | 수정 코드 → 발동 가능 형태로 빌드 | 완성 쾌감 |
| 런타임 에러 | 스킬 발동 중 폭주/역효과 | 긴장감 |
| 메모리 누수 | 마나/기 지속 소모 (해제 안 됨) | 밸런스 위기 |
| 스택 오버플로 | 스킬 중첩 과부하 | 한계 표현 |
| 패치 | 스킬 코드 부분 수정 | 점진적 성장 |
| API 콜 | 타인 스킬 연동/시너지 | 파티 플레이 쾌감 |
| 레거시 코드 | 고대 스킬, 해독 불가 | 미스터리/떡밥 |
| Git 커밋 | 스킬 수정 확정 (롤백 가능) | 선택의 무게 |
| 롤백 | 수정 실패 시 이전 버전 복원 | 역전 기회 |
| 샌드박스 | 스킬 테스트 격리 공간 | 안전한 실험 |

## 3. 서사 기점 설정

### 3-1. 주인공의 과거/배경

- **최고점**: 없음. 안강해는 한 번도 정상에 서본 적이 없다. IT에서 만년 2차 협력사. 헌터 사회에서 F급. 10년 전부터 "미친놈" 취급. 이것이 핵심이다 -- 정상에서 떨어진 게 아니라, **바닥에서 시작해 바닥을 파고 내려간 남자가 처음으로 삽을 발견한다.**
- **좌절/전환**: 대각성으로 전 인류가 상태창을 얻었는데, 10년 전부터 보이던 강해만 F급 판정. "10년을 미친놈으로 살았는데 결국 F급이야?" -- 이중의 모욕. 그러나 그의 상태창은 보급형(v1.0)이 아니라 오리지널(v0.1 베타)이며, 스킬이 코드로 보인다.
- **감정 핵심**: "두 세계 모두에서 하청인 남자"의 자조. 독자는 IT 직장인으로서, 그리고 사회적 약자로서 이중으로 공감한다. 분노가 아니라 체념에 가까운 자조 -- 그래서 더 공감된다.

### 3-2. 서사 동력

- **트리거**: 대각성 1년 후, 강해가 퇴근길에 E급 게이트 돌발 발생 → 비각성자로서 대피 불가 → 죽을 위기에서 v0.1 상태창이 2차 활성화 → 스킬 코드가 "편집 가능"해짐 (스킬 컴파일러 각성)
- **시작 상태**: 대각성 1년 후에도 유일한 F급 비각성자. 중소 SI 2차 협력사 4년차. 정신과 이력 때문에 인간관계 단절. 무능력자 부서(언랭커과) 배정 대기 상태.
- **핵심 능력/자산**: 오리지널 상태창 v0.1 베타 → 스킬 컴파일러 (읽기/편집/컴파일)
- **욕망 이동**: "평범하게 살고 싶다" (EP001) → "이 능력으로 밥은 벌 수 있겠다" (EP005) → "이 시스템은 뭔가 이상하다" (EP040) → "이 시스템의 진실을 알고 싶다" (EP060~)

### 3-3. 핵심 역량 모듈

**3-3-1. 기본 역량 3단계**

| 단계 | 능력 | 범위 | 한계 | 소진 조건 |
|---|---|---|---|---|
| 읽기(Read) | 본인/타인 스킬 코드 시각화 | 시야 내 모든 각성자 | 타인 감정 누수 (군중 두통, 강한 감정 시 의식 끊김) | 다수 동시 읽기 시 두통 → 의식 상실 |
| 편집(Edit) | 본인 스킬 코드 수정 | 본인 스킬만 (타인 읽기 전용) | 전투 중 불가 (빌드 룸에서만) | 장시간 편집 시 집중력 소진, 시야 흐림 |
| 컴파일(Compile) | 수정 스킬 → 실행 가능 형태로 빌드 | 스킬 슬롯 수만큼 | SUCCESS/WARNING/ERROR 분기 | 컴파일 실패 시 원본 유지, 재시도 쿨다운 |

**3-3-2. 밸런스 잠금 4종**

| 잠금 | 메커니즘 | 해제/확장 시점 | 서사적 기능 |
|---|---|---|---|
| 빌드 타임 제약 | 전투 중 코드 수정 불가. 빌드 룸에서만 | EP040~ 부분 해제 (핫픽스: 1줄 수정 가능) | 전투 전 준비의 전략성 확보 |
| 스킬 슬롯 제한 | 동시 컴파일 3개 (초반) | EP015: 4개, EP030: 5개, EP050: 6개 | 선택과 집중의 긴장감 |
| 스킬 등급 캡 | F~D급 자유 수정. C+ 보호 영역(`// PROTECTED`) | EP035~ C급 부분 해제, EP060~ B급 접근 | 스케일 확대의 단계적 관문 |
| 버그 리스크 | 수정 스킬 일정 확률 런타임 에러 | 경험 축적으로 확률 감소, 0은 안 됨 | 먼치킨의 리스크 밸런스 |

**3-3-3. 세부 역량 항목 (15개 이상)**

| # | 역량 | EP 범위 | 활용 방법 | 서사적 기능 |

| 4 | **함수 호출 순서 변경** | EP005~ | 스킬 내 실행 순서 재배치 → 예상치 못한 효과 | 창의적 문제 해결의 쾌감 |

| 8 | **타인 스킬 분석 (코드 리뷰)** | EP006~ | 파티원/적의 스킬 코드를 읽고 약점/시너지 파악 → 전술 제공 | 파티 플레이 가치. 히로인 합류 계기 |
| 9 | **버전 관리 (Git 커밋/롤백)** | EP012~ | 스킬 수정 이력 추적, 실패 시 이전 버전 복원 | 실패해도 되돌릴 수 있다는 안전장치 = 더 과감한 수정 가능 |

| 16 | **에러 메시지 분석** | EP055~ | 시스템이 보내는 에러 메시지의 패턴을 역분석 → 시스템 의도 파악 | 시스템 vs 강해 두뇌전의 소재 |
| 17 | **역컴파일 (다운그레이드)** | EP030~ | 적의 고등급 스킬 코드를 분석하여 약점 도출 → 카운터 빌드 | 대형 빌런전 전략 수립 |
| 18 | **샌드박스 (격리 테스트)** | EP020~ | 위험한 코드 수정을 격리 환경에서 테스트 | 리스크 관리. 신중한 먼치킨 |

**3-3-4. 역량 성장 곡선**

```
EP001~010: 기초 (Read/Edit/Compile + 파라미터 튜닝 + 디버깅)
           "스킬 코드가 보인다" → "수정하면 바뀐다" → "이건 쓸 만하다"

EP011~025: 응용 (리팩토링 + 콤보 + 타인 분석 + 버전 관리)
           "약한 스킬도 조합하면 사기다" → "남의 스킬도 읽을 수 있다"

EP026~050: 확장 (API 연동 + 보호 영역 접근 + 핫픽스 + 역컴파일)
           "혼자가 아니라 팀으로" → "C급 스킬도 건드릴 수 있다"

EP051~080: 심화 (시스템 함수 발견 + 난독화 해제 + 에러 역분석)
           "이 시스템은 뭔가 숨기고 있다" → "농장의 진실"
```

**3-3-5. 약점: 타인 감정 누수**

| 상황 | 증상 | 심각도 |
|---|---|---|
| 평시 (소수) | 미세한 감정 잡음 | 무시 가능 |
| 군중 (10명+) | 두통, 집중력 저하 | 코드 읽기 방해 |
| 강한 감정 (공포/분노/광기) | 감정 동조, 판단력 상실 | 전투 중 치명적 |
| 과다 노출 (대형 레이드 등) | 의식 끊김, 수분~수시간 기절 | 행동 불능 |

서사적 기능: 솔플 먼치킨이되 대규모 전투에서는 약점이 있다. 파티원(특히 히로인)의 존재 이유 = 감정 차단/보호 역할.

## 4. 감정적 동인 설계

### 4-1. 핵심 동인: "두 세계 모두에서 하청인 남자"

IT 하청 구조와 F급 헌터 차별이 정확히 겹친다. 안강해는 이 구조적 유사성을 자조적으로 인식한다.

| IT 현실 | 헌터 사회 | 강해의 자조 |
|---|---|---|
| 원청 식당 이용 불가 | 길드 시설 등급별 접근 제한 | "밥값 차별은 차원을 초월하는군" |
| "협력사분" 호칭 | "F급" 호칭 | "호칭이 바뀌어도 뉘앙스는 같다" |
| 성과 원청 귀속 | 드롭 아이템 상납 | "내가 잡았는데 네 거라고?" |
| 프로젝트 종료 후 퇴출 | 레이드 직전 F급 교체 | "필요할 때만 부르는 건 여기도 마찬가지" |

### 4-2. 독자 감정이입 극대화 전략

**EP001~003에서 반드시 배치할 감정 트리거**:

1. **부당한 무시**: 10년 전부터 상태창이 보였는데 "미친놈" 취급. 대각성 후에도 F급. "10년을 앞서 봤는데 결국 꼴찌"
2. **이중 착취**: IT 파견사에서 야근 → 퇴근 후 무능력자 부서 소집 → 양쪽에서 착취
3. **능력의 자각**: 남들은 스킬 카드(아이콘 + 설명)만 보는데, 강해에게는 코드가 보인다. 처음엔 "또 내가 이상한 건가" → "잠깐, 이건 버그가 있는데?" → 수정 → 효과 → "이건... 되는데?"
4. **자조의 매력**: 영웅적 결의 대사 절대 금지. "세계를 구하겠다"가 아니라 "일단 이번 달 월세부터". 독자는 현실적 동기에 더 공감한다.

### 4-3. 감정 위상 변화 (1부 전체)

```
EP001~005: 체념 → 자각 → "이건 쓸 만하다"
EP006~020: 조심스러운 활용 → 첫 대형 사이다 → "나도 할 수 있다"
EP021~050: 자신감 → 조직적 위협 → "이 구조를 바꿔야 한다" (but 자조 유지)
EP051~080: 의심 → 시스템 진실 → "...이건 단순한 게 아니었어"
```

## 5. 셀링포인트

### 5-1. 핵심 차별화 (R5 반영)

**시장 빈 공간**: 코딩 소재 헌터물 대형 히트작 부재 (R4). "프로그래밍 메타포를 장식이 아닌 핵심 메커니즘으로 활용한 작품"이 없다.

**차별화 5대 축** (R5 빈 공간 분석):

| 축 | 기존작 | skill-compiler |
|---|---|---|
| 스킬 성장 방식 | 레벨업/아이템/스킬 흡수 | **코드 편집 (디버깅/리팩토링/콤보)** |
| 시스템과의 관계 | 순응(나혼렙) 또는 메타(전독시) | **개발자가 코드를 읽고 수정하는 해킹** |
| IT 직업 활용 | 장식적 배경 | **능력 시각화 자체가 IT 코드** |
| 주인공 톤 | 3인칭 무감정 또는 코미디 | **1인칭 자조/냉소 + 사이다** |
| F급 성장 공식 | 숨겨진 재능/각성 | **재능 없음. 코드를 읽는 눈 + 4년차 디버깅 근성** |

### 5-2. 주요 쾌감 요소

| 쾌감 유형 | 메커니즘 | EP 배치 |
|---|---|---|
| **최적화 쾌감** | 약한 스킬의 코드를 수정하여 강화 → "이 코드 한 줄로?" | 매 EP (핵심 루프) |
| **등급 역전 사이다** | F급이 A급 스킬 콤보 시전 → 주변 경악 | EP010, 020, 035, 050, 075 (대형 사이다 포인트) |
| **탐정 쾌감** | 스킬 코드 분석 → 숨겨진 조건/버그 발견 | EP당 1회 이상 |
| **자조 웃음** | IT 하청 현실 + 헌터 차별의 자조적 관찰 | 매 EP (톤 유지) |
| **시스템 미스터리** | 코드 속 숨겨진 함수, 의념 수확 흔적 | EP020부터 점진 배치 |
| **파티 시너지** | 타인 스킬 분석 → 최적 전술 제공 → 팀 승리 | EP015~ |

### 5-3. 익숙함:새로움 비율 (7:3)

**익숙함 70%**:
- F급 → 먼치킨 성장 공식
- 현대 한국 + 게이트/던전/길드 체계
- 솔플 → 파티 → 길드전 스케일 확대
- 5대 길드 간 갈등, 협회 부패
- 사이다 카타르시스

**새로움 30%**:
- 스킬 = 수정 가능한 코드 (프로그래밍 메타 메커니즘)
- 시스템 = 의념 수확 농장 (시스템 해부)
- IT 하청 + F급 차별의 구조적 겹침 (이중 소외 자조)

## 6. 적대 구조 (3중 레이어)

### 6-1. 갈등 4단계 계층 (R6 반영)

| 서사 단계 | 갈등 레벨 | 적대자 유형 | IT 모티프 | 독자 감정 |
|---|---|---|---|---|
| EP001~020 | **개인** | A급 헌터의 무시, 길드 강제 영입 | 상사/PM의 부당 지시 | 공감 → 사이다 |
| EP021~050 | **조직** | 대형 길드 기술 탈취, 협회 등급 조작 | 원청 기술 독점, 갑을 관계 | 분노 → 통쾌 |
| EP051~080 | **사회** | 5대 길드 vs 솔로/소형 전쟁, F급 퇴출 정책 | 플랫폼 독점 vs 중소 개발사 | 구조적 분노 |
| EP080~ (2부) | **시스템** | 의념 수확 농장 폭로, 0층 대면 | AI/시스템의 인간 지배 | 존재론적 충격 |

### 6-2. 1층 빌런 (인간)

5대 길드 중 1~2개의 길드 마스터/간부가 얼굴 빌런.

| 빌런 유형 | IT 모티프 | 서사 기능 |
|---|---|---|
| 재벌 계열 길드 마스터 | 대기업 원청의 하도급 착취 | 사이다 대상. 코드 탈취 시도 → 역관광 |
| 협회 기술국장 | 플랫폼 알고리즘 조작 | 등급 시스템 편향 폭로. F급인 이유의 실마리 |

**빌런 다층성**: 초반 빌런(길드 간부)이 시스템의 피해자이기도 했음이 나중에 밝혀진다. "시스템에 순응한 자" vs "시스템에 저항한 자"의 도덕적 회색 지대.

### 6-3. 2층 빌런 (시스템)

| 발견 단계 | 내용 | EP 범위 |
|---|---|---|
| 무지 | 시스템을 선물로 인식 | EP001~019 |
| 의심 | 코드에서 이상한 함수 발견 (`harvest_intention` 등) | EP020~039 |
| 인지 | 시스템이 각성자의 의념을 수확함을 파악 | EP040~059 |
| 대응 | 수확 함수 우회/변조 코드 작성 → 시스템이 강해를 위험 요소로 인식 | EP060~080 |

### 6-4. 0층 빌런 (설계자)

시스템을 만든 존재. 2부 이후 떡밥. 1부에서는 존재 암시만.
- "이전에 코드를 본 자가 있었으나 '시스템 오류'로 사망 처리" (EP050~ 떡밥)
- 강해가 F급인 진짜 이유: 시스템이 의도적으로 코드 편집 능력자의 등급을 F로 분류 → 위협 관리

## 9. 기존 작품 대비 포지셔닝 (R5 반영)

### 9-1. 포지셔닝 맵

```
     [시스템 순응] ←─────────────────────→ [시스템 해체]
          |                                    |
    나혼렙 *          최강시스템 *         전독시 *
          |                                    |
          |         SSS급자살 *                |
          |                                    |
    [단순 먼치킨]                         [서사 깊이]
          |                                    |
    템빨 *                                     |
          |                                    |
          |      ★ skill-compiler ★            |
          |      (시스템 해킹 x 먼치킨 x        |
          |       코딩 메타 x 자조)             |
```

**목표 위치**: 시스템 해체 축에서 전독시에 가깝되, 먼치킨 사이다 축에서 나혼렙에 가까움. **지적 쾌감 + 먼치킨 쾌감의 교차점.**

### 9-2. 상세 비교

| 비교 작품 | 유사점 | 차별점 |
|---|---|---|
| 나 혼자만 레벨업 | F→S급 극적 성장, 솔플 먼치킨, 시스템 수혜 | 시스템이 적대자(농장). 1인칭 자조. 코드 편집으로 성장 |
| 전지적 독자 시점 | 시스템 해부, 메타적 접근, 깊은 세계관 | 문학 메타가 아닌 프로그래밍 메타. 먼치킨 사이다 비중 높음 |
| 템빨 | F급 먼치킨, 빠른 전개 | 아이템 수집이 아닌 코드 편집. 서사 깊이 추가 |
| SSS급 자살 헌터 | 독창적 능력, 코미디+진지 | 남의 스킬 복사가 아닌 자기 스킬 수정. IT 직업 연동 |
| 최강의 시스템 | 시스템 보조 성장 | 시스템 순응이 아닌 시스템 해킹. 코드가 보이는 독자적 관점 |
| 버그 플레이어 | 게임 버그 활용 | 비의도적 결함 활용이 아닌 의도적 코드 수정 |

### 9-3. 차별화 빈 공간 5개 (R5)

| # | 빈 공간 | skill-compiler 활용 |
|---|---|---|
| 1 | 프로그래밍 메타포의 체계적 활용 | 스킬 = 코드. 디버깅/리팩토링/컴파일이 전투의 핵심 |
| 2 | 시스템을 "개발자 시점"으로 해부 | 강해가 시스템 소스코드를 읽고 의도(의념 수확)를 파악 |
| 3 | IT 하청 + F급 차별의 구조적 대비 | 두 세계 모두에서 "하청"인 남자의 이중 소외 자조 |
| 4 | 약한 스킬의 코드 최적화 | E급 스킬 리팩토링 → B급 성능. 등급은 그대로인데 성능이 다름 |
| 5 | 시스템 = 의념 수확 농장 | 구체적 시스템 비판. 각성자 = 가축, 스킬 = 사료, 의념 = 수확물 |

## 10. 미디어 믹스 확장성

### 10-1. 웹툰화 적합도: ★★★★☆ (높음)

- 코드 블록 시각화 = 독특한 비주얼 연출 가능 (HUD/IDE 화면 오버레이)
- 스킬 코드 수정 장면 = 타이포그래피 + 이펙트 결합
- 컴파일 결과(SUCCESS/WARNING/ERROR) = 색상 연출 (녹/황/적)
- 콤보 발동 = 코드 라인이 빛나며 연결되는 비주얼

### 10-2. 드라마화 적합도: ★★★★★ (매우 높음)

- 현대 한국 배경 = 별도 세트 불필요
- IT 직장인 일상 = 높은 현실성, 공감대
- 스킬 코드 = AR/VR 오버레이로 시각화 가능
- 직장 드라마 + 히어로물의 결합 = 넓은 시청자층

### 10-3. 핵심 시각화 장면 3개

1. **빌드 룸에서 코드 편집**: 반투명 코드 블록이 허공에 떠오르고, 강해가 손가락으로 코드를 수정하는 장면. IDE 화면이 현실 공간에 투사된 느낌
2. **콤보 발동**: 3개 스킬의 코드 라인이 각각 다른 색으로 빛나며 하나로 합쳐지는 순간 → 폭발적 이펙트
3. **시스템 코드 발견**: 이상 던전에서 세계의 "소스코드"가 벽면에 흘러내리는 장면. Matrix 오마주이되, 한국어/코드 혼합
