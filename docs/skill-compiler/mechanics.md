# 메커닉 — 스킬 코드 시스템

> 본 문서는 능력의 시각·실행·등급 변환 사양을 정의.
> 큰 설계에서 등급별 코드 복잡도 사양·견본 코드 추가.

---

## 1. 시각화 (강해의 시야)

### 1-1. 보급형 vs 강해

| 보급형 (일반인) | 강해 (오리지널) |
|---|---|
| 스킬 카드: 아이콘 + 1~3줄 설명 | 코드 블록 (함수·변수·조건문) |
| 쿨다운: "10초" | `cooldown: 10s` |
| 부가 효과: "출혈" | `if (target.armor < 0.3) { apply_bleed(3s); }` |

### 1-2. 본문 표현 가이드

| 단위 | 빈도 | 예시 |
|---|---|---|
| 큰 코드 블록 (5~15줄) | EP당 1~2회 | 빌드 룸 신, 새 스킬 분석 신 |
| 인라인 코드 | 자유 | "`damage = 0.8 * STR`이라는 한 줄이 보였다" |
| 주석 처리 (보호 영역) | 등장 시 강조 | `// PROTECTED — DO NOT MODIFY` |

### 1-3. 시각화 톤 가이드

- 강해의 1인칭 독백에서 코드는 **기능적 묘사** + **자조 한 줄**로 마무리
- 예: "skill_id: F_Slash. 코드는 다섯 줄짜리 파편. ... 이런 게 D급이다."
- 코드 자체를 화려하게 미화 X. 담담하게.

## 2. 편집 사이클

```
1. 빌드 룸 진입 → 자기 스킬 코드 노출
2. 수정 (변수·조건문·함수 호출 순서·인자)
3. 컴파일 → 빌드 결과
   - SUCCESS: 즉시 사용 가능
   - WARNING: 사용 가능하나 부작용 알림
   - ERROR: 컴파일 실패, 원본 유지
4. 던전 진입 → 실전 사용
5. 후속 안정화 (반복 테스트 → 신뢰도 ↑)
```

### 2-1. 컴파일 결과 시각

- SUCCESS: 깔끔한 한 줄 메시지
- WARNING: 노란 글자 + 부작용 명세 (예: "스태미나 -20%")
- ERROR: 빨간 글자 + 라인 넘버 (디버그 필요)

---

## 3. 버그 분류

| 버그 | 증상 | 빈도 |
|---|---|---|
| 런타임 에러 | 발동 즉시 실패 | 수정 직후 흔함 |
| 역소환 | 본인 데미지 | 잘못된 타겟 변수 |
| 침묵 | 일정 시간 스킬 잠금 | 무한 루프 / 자원 고갈 |
| 의도치 않은 부작용 | 의외의 시너지 또는 패널티 | 가장 흥미로운 케이스 — 신규 빌드 발견 |

---

## 4. 등급 변환 모델

| 등급 | 코드 복잡도 | 외부 의존성 |
|---|---|---|
| F~E | 함수 1~2개, 인자 1~2개, 조건문 X | 없음 |
| D | 함수 2~3개, 조건문 1~2개 | 없음 |
| C | 라이브러리 호출, 보호 영역(주석 처리) 일부 | 약함 |
| B | 외부 의존성 (정령·계약·신성) | 보통 |
| A | 다중 의존성, 의념 결합 흔적 | 강함 |
| S+ | 시스템과 직접 통신 인터페이스 | 매우 강함 |

→ 강해의 강점: F~D 자유 수정 + 콤보. C+ 부분 수정만 (보호 영역 우회 X 초반).

---

## 5. 사기 콤보 (브리프 §3)

> "약한 D급 스킬 세 개를 조합하면 A급 콤보가 된다"

### 5-1. 콤보 조건

- 발동 타이밍 동기화 (트리거 변수 공유)
- 자원 풀 분리 (한 스킬 자원 고갈로 다른 스킬 죽이지 않게)
- 부작용 상쇄 또는 시너지

### 5-2. 콤보 견본 (큰 설계에서 확정)

- D급 슬래시 + D급 약화 디버프 + D급 출혈 → A급 다중 출혈 콤보
- D급 둔속 + D급 빙결 시드 + D급 카운터 → 광역 카운터 빌드
- (등 5~10개 견본 큰 설계에서 확정)

---

## 6. 타인 능력 분석 (메인 히로인 신호탄)

- 강해는 타인 스킬 코드를 **읽을 수만** 있음 (편집 X)
- 분석 결과로 사용법·약점·시너지를 알려줄 수 있음
- 메인 히로인의 단일 강력 스킬 → 강해가 코드 본 후 응용법 알려줌 → 그녀가 따라옴 (EP015~020)
