## 1. 시점·서술

- **1인칭 (안강해)**
- **자조·냉소** → 갑작스런 우월감 대비 (브리프 §10)
- **평서문 융합** — 작은따옴표 생각풍선 폐기 (narrative-style v2 §1·§2)
- **외부 인물 내면 직접 서술 X** — POV 강해 밀착
- **"그렇게 생각했다" 류 명시 진입은 가끔만** — 기본은 평서문 보이스 융합

### 1-1. 1인칭 자조 톤 견본

```
"공식 스킬 셋 — F_Slash, F_Block, F_Heal. 다섯 줄짜리 파편 셋이 전부였다. 이런 걸 들고 던전을 가라고? 협회 측정실의 측정관도 같은 표정이었다. 한숨까지 비슷했다."
```

- 짧은 문장 호흡
- 코드 → 자조 한 줄 → 외부 반응 한 줄 패턴
- 미화 어휘 X ("파편", "겨우", "이런 거" 등 절제)

### 1-2. 미래 시점 예고 (v2 §8)

- 가끔 "그때까지만 해도 ~ 거라고 생각했다" 클로징 허용
- 1막에 1~2회

## 2. 사이다 사이클 (브리프 §10)

```
무시받음 → 처발림 → 충격 받는 주변 반응 → (강해의 무덤덤 마무리)
```

- 짧게 반복. 한 EP당 1~2 사이클
- 권위적 적대(길드/협회) 처발리는 그림 자주
- **영웅적 결의 대사 절대 금지** — 행동만 영웅, 독백은 자조

### 2-1. 사이다 패턴 예시

| 패턴 | 예 |
|---|---|
| 등급 측정 | F급 인증 → 강해 무덤덤 / 측정관 어쩔 줄 모름 |
| 길드 영입 | 길드 마스터 거만 → 거절 → 마스터 당황 |
| 던전 클리어 | 솔플 클리어 보고 → 공격대장 침묵 |

## 4. 분량·절단신공

- **EP당 5,000~7,000자** (공백 포함, 리더 카운트)
- **절단신공 우선** — 5,000자 도달 직전 위기 또는 보상 직전에서 끊기
- 5천 자에서 못 왔으면 **8천 자까지 늘려서라도** 그 지점에서 끊기 (브리프 §10)
- 클리프행어 단문 클로징 (v2 §5)

## 5. 코드 블록 표현 (메커닉 직결)

- 큰 코드 블록(5~15줄): 회당 1~2회 — 마크다운 ``` 블록
- 인라인 코드: 자유 — `damage = 0.8 * STR` 등
- 주석 처리(보호 영역): 등장 시 강조 — `// PROTECTED`
- 코드는 **능력 시각화 자체** → IT 비유 횟수 제한 X
- 단, 작품 외 일반 비유(인생을 race condition에 비유 등)는 절제 — EP당 0~1회

### 5-1. 코드 + 자조 결합 견본

```python
def F_Slash(target):
    damage = 0.8 * STR
    apply_damage(target, damage)
```

> 다섯 줄. 끝.
> 이게 D급이라고 했다. 아니, 정확히는 D급 중에서도 가장 약한 축.

## 7. 1~5화 의무 사항 (브리프 §10)

- **1화 안에 후크 박힘** — 모두에게 상태창이 열린 날, 강해의 무덤덤 회상부터
- **5화 안에 각성 + 첫 사이다** — 자기 등급 위 던전 솔플 클리어
- **프롤로그 짧게 또는 생략** — 본편 1화에서 즉시 후크
- 5천 자 시점 절단신공 위치를 매 화 사전 정해두고 비트 시트 짤 것

---

## 8. 운영 전략 (문피아 기준 — 브리프 §10)

- 비축분 EP001~EP010~020 쌓고 시작
- 매일 1편 임팩트 (조회 20×1편 > 조회 10×3편)
- 인기 작가 포진한 자정 시간대 피하기 (출퇴근/새벽 노릴 것)
- 25화 시점 투베 200위 안 못 들면 손절 검토 (PM 마인드)
