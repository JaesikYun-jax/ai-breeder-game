## A. 스탯 체크 시스템

### 체크 방식: 순수 임계값 (랜덤 없음)

스탯이 임계값 이상이면 성공, 미만이면 실패. 주사위/RNG 없음.

**설계 이유**:
- 비주얼노벨에서 랜덤 실패는 좌절감만 줌
- 루프물의 핵심 쾌감 = "이번 루프에서 쌓은 스탯으로 전에 못 넘던 관문 돌파"
- 플레이어가 성장을 투명하게 체감해야 함

### 3단계 결과 구조

각 스탯 체크는 최대 3단계 결과를 가짐:

| 결과 | 조건 | 예시 (STR 체크) |
|------|------|----------------|
| **성공** | stat >= success 임계값 | STR >= 5 → 완승 |
| **부분 성공** | partial <= stat < success | STR 3~4 → 고전 끝 승리 |
| **실패** | stat < partial (또는 success) | STR < 3 → 패배/사망 |

```typescript
interface StatCheck {
  stat: "STR" | "INT" | "CHA" | "PER" | "FAT";
  success: number;        // 이상이면 성공
  partial?: number;       // 이상이면 부분 성공 (생략 시 = success 미만은 전부 실패)
  successNext: string;    // 성공 시 이동 노드 ID
  partialNext?: string;   // 부분 성공 노드 ID
  failNext: string;       // 실패 노드 ID
  successEffects?: GameEvent[];
  partialEffects?: GameEvent[];
  failEffects?: GameEvent[];
}
```

### 아젤리아 Ch1 스탯 체크표

| 노드 | 체크 | 성공 | 부분 | 실패 |
|------|------|------|------|------|
| 007-D 왕의 침실 | PER >= 4 | 목격 | — | 접근 불가 |
| 020 마을 전투 | STR >= 5 or INT >= 5 | 완승 | 3~4: 고전 | < 3: 사망#1 |
| 030 에리나 비밀 | 호감 >= 15 | 발동 | — | 스킵 |
| 040-A 전장 | STR >= 8 | 영웅 귀환 | 5~7: 부상 | < 5: 사망#2 |
| 042 밀서 관찰 | PER >= 7 | 완전 목격 | 5~6: 감지만 | < 5: 놓침 |
| 043 궁전 방어 | CHA >= 6 & STR >= 4 | 완전 방어 | 하나만 충족: 부분 | 둘 다 미달: 사망#3 |
| 011d 정면 대결 | CHA >= 8 or INT >= 8 | 대면 성공 | 6~7: 후퇴 | < 6: 사망#4 |

**설계 의도**: 루프1(퀴즈 초기 스탯 ~20점)에서는 높은 임계값(8+) 통과 불가. 2~3루프 축적 후 돌파 가능.

## B. 영혼 귀속 시스템

### B-1. 영혼 스탯 (Soul Stats)

스탯은 영혼에 연결되어 루프 간 누적됨.

**누적 공식**:
```
루프 시작 시 실효 스탯 = 퀴즈 초기값 + soulStats
루프 종료(사망/엔딩) 시:
  soulStats[X] += floor(해당 루프에서 획득한 X × 0.5)
```

**상한**: 실효 스탯 최대 15 (soulStats 자체도 12 상한)

**예시**:
```
루프1: 퀴즈 STR=3. 루프 중 STR+4 획득. 사망.
  → soulStats.STR += floor(4 × 0.5) = 2. 누적 soulStats.STR = 2
루프2: 실효 STR = 3(퀴즈) + 2(soul) = 5. 루프 중 STR+3 획득. 사망.
  → soulStats.STR += floor(3 × 0.5) = 1. 누적 soulStats.STR = 3
루프3: 실효 STR = 3 + 3 = 6. → 020 마을 전투(STR 5+) 여유 있게 돌파!
```

### B-2. 영혼 귀속 아이템 (Soul Items)

특정 이벤트 경험 + 사망 조합으로 "영혼에 각인"되는 아이템. 루프 후에도 유지.

**핵심 규칙**: 살아서 끝내면 일반 인벤토리(루프 시 소멸). **사망 시에만 영혼에 각인**하여 영구 보존. → 사망에 가치 부여.

```typescript
interface SoulItem {
  id: string;
  name: string;
  description: string;
  acquiredLoop: number;      // 획득한 루프
  acquiredRegion: string;
  narrativeNote: string;     // "영혼에 각인된 이유"
  opensPath?: string[];      // 열리는 노드/조건 키
}
```

## C. 수확체감 시스템

### 원칙

같은 이벤트를 반복해도 스탯이 무한히 오르지 않음. 영혼 스탯에 반영되는 양이 줄어듦.

### 체감 공식

```
실제 보상 = floor(기본 보상 × 체감 계수)

체감 계수:
  1회째 완료: 1.0 (100%)
  2회째 완료: 0.5 (50%)
  3회째+ 완료: 0.0 (0%)
```

### 추적

`soulData.eventLog: Record<string, number>` — key: 이벤트 ID, value: 완료 횟수

### 적용 예시

| 이벤트 | 기본 보상 | 1회째 | 2회째 | 3회째+ |
|--------|----------|-------|-------|--------|
| 020 전투 승리 | STR+2 | +2 | +1 | +0 |
| 030 에리나 위로 | CHA+2 | +2 | +1 | +0 |
| 005 축복 수락 | FAT+2 | +2 | +1 | +0 |
| 007-B 도서관 | INT+1 | +1 | +0 | +0 |

### 체감 면제 (항상 100%)

다음은 1회성이므로 자연적으로 면제:
- 진실 플래그 해금 (`truth_*`)
- 멜로디 파편 획득
- 영혼 귀속 아이템 획득
- 사망 경험 메타 플래그

### 새로운 이벤트로 우회

같은 노드(020 마을)라도 **다른 해결 방법**은 별개 이벤트:
- `020_combat_victory` (전투 승리) — 3회째+ 체감됨
- `020_monster_release` (마물 해방, 루프5+ INT 10) — 완전히 새 이벤트, 풀 보상

→ 반복 시 새로운 접근법을 찾도록 유도.

### UI 표현

| 상태 | 표시 |
|------|------|
| 풀 보상 | 정상 표시 `[STR +2]` |
| 체감됨 | 회색 + 취소선 `[~~STR+2~~ STR+1]` |
| 소진됨 | `(이 경험에서 더 이상 얻을 것이 없다)` |
| 면제 보상 | 금색 테두리 `[영혼에 새겨진 경험]` |

## D. 루프별 콘텐츠 변화 체계

### 오버라이드 방식

루프별로 전체 JSON을 복제하지 않음. **변경된 노드만** 오버라이드 파일에 넣고 StoryEngine이 deepMerge.

```
src/data/story/azelia/
├── chapter1.json           # 베이스 (1회차)
├── chapter1_loop2.json     # 2회차 변경분만
├── chapter1_loop3.json     # 3회차+ 변경분만
└── chapter1_loop5.json     # 5회차+ 변경분만
```

```
로드 우선순위: loop5 > loop3 > loop2 > base
(높은 루프 오버라이드가 있으면 그것 사용)
```

### 루프별 변화 유형

| 루프 | 변화 유형 | 예시 |
|------|----------|------|
| 1회차 | 없음 (베이스) | 표준 클리셰 경험 |
| 2회차 | 기시감 독백 삽입 | "(이 장면... 본 적 있는 것 같다)" |
| | soul item 기반 새 선택지 | soul_erina_ribbon → "이 리본을..." |
| | 사망 기억 기반 텍스트 변경 | "(이번에는 그렇게 당하지 않는다)" |
| 3회차+ | 메타 플래그 기반 대폭 분기 | truth_azelia → "대신관의 정체를 안다" |
| | 높은 스탯으로 새 체크 돌파 | STR 8+ → 전장 완승 |
| | NPC 관계 빠른 발전 | soul item으로 초기 호감 부스트 |
| 5회차+ | 스킵 옵션 | "이미 아는 장면이다" → 핵심부 직행 |
| | 냉소적/효율적 선택지 | "또 이 의식이야..." |
| | 고급 해결 루트 | INT 10+ → 전투 없이 마물 해방 |
| 10회차+ | 대사 전문 변조 | 모든 텍스트에 피로감 반영 |
| | 시간 정지 능력 선택지 | "[시간의 힘] 잠시 멈춘다" |

### 필수 사망 (Required Deaths)

특정 메타 플래그는 **사망을 통해서만** 해금:
- `mordeus_execution_seen`: 빛의 사슬 대응법 없이 모르데우스 최종 대면 불가
- `pillar_direct_danger`: 기둥 안전 접근법 없이 루트 F 진입 불가
- 이를 통해 "죽음이 진보"라는 루프물 핵심 테마 강화

## F. New Game+ 루트 맵

### 루프별 최적 경로

```
=== 루프 1: 탐색 루프 (사망 예정) ===
001(손잡기 CHA+1) → 003(에리나 CHA+1) → 005(축복 FAT+2)
  → 007(정원+도서관+막사) → 010(수락 CHA+2) → 020(고전승 STR+1)
    → 040-A(전장) → [사망#2: 전사]
수확: soulStats CHA+2 FAT+1, imperial_encounter, soul_light_shard

=== 루프 2: 정보 수집 (사망 가능) ===
001(기시감, soul_light_shard 반응) → 003(마르코 PER+1)
  → 005(거절 INT+1 CHA+1) → 007(도서관+왕의침실 PER체크 통과)
    → king_medicine_seen → 010(대면시도) → [사망#4: 처형]
수확: mordeus_execution_seen, soulStats INT+1 PER+1

=== 루프 3: 진실 루트 (엔딩 B 달성) ===
001(메타: "대신관을 조심해", 에리나+15) → 003(에리나 심화)
  → 005(질문 INT+2) → 007(전구역 + 지하성소)
    → 030(erina_confession) → 042(밀서 PER 7+ 달성)
      → 050 루트 B(진실의 추적자) → 챕터 2!
수확: truth_azelia, soul_mordeus_seal

=== 루프 5+: 효율 루트 ===
001(스킵→007) → 007(지하성소 직행) → 증거 확보
  → 010(CHA 10+ 왕 설득) → 050 루트 D(왕좌의 전복)
```
