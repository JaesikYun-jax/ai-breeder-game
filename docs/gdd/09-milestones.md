# 09. 개발 마일스톤

---

## Phase 0: 프로토타입 (MVP)
**목표**: 하나의 지역(아젤리아)으로 핵심 게임 루프 검증

### 기술 기반
- [ ] Vite + Phaser 3 프로젝트 셋업 ✅
- [ ] StoryEngine 구현 (노드 그래프 기반)
- [ ] FlagManager 구현 (일반 플래그 + 메타 플래그)
- [ ] LoopManager 구현 (회귀 횟수, 방문 기록)
- [ ] JSON 스토리 데이터 로더
- [ ] localStorage 기반 세이브/로드

### 씬 구현
- [ ] BootScene (에셋 로딩)
- [ ] MenuScene (새 게임 / 이어하기)
- [ ] StoryScene (텍스트 표시 + 선택지)
- [ ] UIScene (HUD 오버레이)

### 스토리 콘텐츠
- [ ] 프롤로그 (현대 사망 → 전이)
- [ ] 성격 퀴즈 5문항
- [ ] 아젤리아 챕터 1 (최소 20개 노드)
- [ ] 사망 → 회귀 시퀀스
- [ ] 아젤리아 일반 엔딩 1개

### 연출
- [ ] 텍스트 타이핑 효과
- [ ] 선택지 UI (일반 + 메타)
- [ ] 페이드 인/아웃 전환
- [ ] 사망/회귀 시 화면 이펙트 (기본)

### 완료 기준
> 아젤리아 챕터 1을 처음부터 끝까지 (또는 사망) 플레이 가능.
> 사망 후 회귀하여 2회차에서 메타 선택지 등장 확인.

- [ ] 크로스 지역 메타 이벤트 1개

- [ ] BGM 레이어 시스템 (기본)

### 콘텐츠

- [ ] 마르코 교역 시스템
- [ ] 검은 로브 여인 등장 (9개 지역)
- [ ] 멜로디 파편 시스템 (9개 수집)
- [ ] 관계 엔딩 9개

- [ ] 10번째 멜로디 삽입

## Phase 3: 트루 엔딩 + 폴리싱
**목표**: 전체 엔딩 트리 완성 + 게임 품질 최종 점검

### 콘텐츠
- [ ] 각 지역 챕터 2 (9개)
- [ ] 트루 엔딩 3종 (복원/파괴/공존)
- [ ] 특수 엔딩 3종 (고독/관측자/각성실패)
- [ ] 크로스 지역 이벤트 6개
- [ ] 진실 플래그 9개 해금 루트

### 시스템
- [ ] 트루 엔딩 조건 판정
- [ ] 회귀 20회차+ 연출 풀패키지
- [ ] 시간 정지 능력 (20회차 해금)
- [ ] 기둥화 경고 시스템 (30회차)

### 폴리싱
- [ ] SFX 전체 삽입
- [ ] 접근성 옵션 (화면 이펙트 OFF, 텍스트 크기 등)
- [ ] 모바일 터치 최적화
- [ ] 성능 최적화 (에셋 lazy loading)
- [ ] QA: 전체 분기 경로 검증 (고아 노드 없는지)

## 기술 부채 / 리스크

| 리스크 | 대응 |
|--------|------|
| 스토리 분기 폭발 | 노드 재사용 + 플래그 기반 분기로 경량화 |
| 에셋 용량 | 단계별 로딩 + WebP 포맷 |
| 모바일 성능 | Phaser Scale FIT + GPU 렌더링 |
| 세이브 데이터 손실 | localStorage + 선택적 클라우드 백업 |
| 콘텐츠 일관성 | Red/Blue 팀 리뷰 + 기획자 최종 승인 |
