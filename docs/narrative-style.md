> **확정**: 2026-04-25
> **적용 범위**: 모든 신규 챕터·신규 프로젝트. 진행분 소급 X.

## 1. 단락 길이 & 클러스터링

### 1.1 단락 길이 — 1~3줄 기본형
- 거의 모든 단락이 **1~3줄**. 모바일 스크롤 리듬 = 짧은 호흡 반복.
- "한 줄 단락 5~7회 제한" 같은 상한 룰 **폐기**. 짧은 단락이 기본이다.
- **단, 4줄 초과 검출 룰 없음** — 의미상 한 호흡이면 4~5줄도 허용.

### 1.2 단락 클러스터링 — 무의미한 줄바꿈 방지

같은 호흡·같은 인물의 연속 사고·행동·감각은 **빈 줄 없이 단일 줄바꿈(`\n`)으로 묶어** 한 단락 클러스터로 표현. 빈 줄(`\n\n`)은 호흡·시점·대사 전환에서만.

**좋은 예** (4문장 한 클러스터):
```
오늘로 정확히 100번째.
이제는 눈 감고도 외울 수 있는 문구였다.
박지훈은 캔맥주를 한 모금 들이켰다.
그러나 탄산은 밍숭맹숭하기만 했다.
```

**나쁜 예** (모든 문장 사이 무의미한 빈 줄):
```
오늘로 정확히 100번째.

이제는 눈 감고도 외울 수 있는 문구였다.

박지훈은 캔맥주를 한 모금 들이켰다.

그러나 탄산은 밍숭맹숭하기만 했다.
```

**빈 줄 삽입 기준**:
- ✓ 대사 ↔ 서술 전환
- ✓ 씬 내부 시간·장소·시점 전환
- ✓ 의도적 정적·여백 강조 (단발 클로징, 충격적 한 줄 등)
- ✗ 같은 인물의 연속 행동·사고·감각 묘사 사이
- ✗ 한 사건·한 호흡 내 인과 연결 사이

**자주 실수하는 클러스터 케이스 (2026-04-25 피드백 누적)**:
- ❌ "어떤 자국을 발견했다 / [빈 줄] / 인물이 숨을 들이켰다" — 발견과 즉각 반응은 한 호흡, 빈 줄 X
- ❌ "긴 대사·욕했다 / [빈 줄] / 그러다 멈췄다" — 같은 인물의 연속 행동, 빈 줄 X
- ❌ "어떤 사실을 인지했다 / [빈 줄] / 같은 인물이 그 인지에 따라 마음을 풀었다" — 인지와 반응은 한 호흡, 빈 줄 X
- ❌ "더 머물기 어려웠다 / [빈 줄] / 문을 닫고 올라갔다" — 의지와 행동, 빈 줄 X
- 점검 방법: 빈 줄 양옆 두 단락이 **같은 인물의 같은 호흡**인지 자문

## 2. 내면 = 평서문 서술자 보이스에 융합

**핵심 원칙**: **POV 캐릭터 밀착 시점**. 1인칭이든 3인칭 자유간접화법이든 동일하게 적용.

- 형식이 3인칭이라도 내용은 **POV 캐릭터(주로 주인공)만 알 수 있는 내면**까지 직접 진입한다.
- 예: "그도 어렴풋이 느끼고 있었다", "정말로 괴이한 일이었다" — 형식은 3인칭, 내용은 1인칭 인지.
- POV 캐릭터 **외부 인물의 심리 직접 서술 금지** (대사·표정·행동으로만 노출).
- `'작은따옴표 생각풍선'` **전면 폐기**.

내면은 다음 4가지 방식으로 표현한다:

### 2.1 감각 = 심정으로 치환
```
그러나 탄산은 밍숭맹숭하기만 했다.
등까지 들러붙은 뱃가죽에서 꼬르륵거리는 소리가 나고 있었다.
```

### 2.2 명시적 진입은 짧은 1줄로
```
박지훈은 그렇게 생각했다.
그도 어렴풋이 느끼고 있었다.
정말로 괴이한 일이었다.
```

### 2.3 미래 시점 예고
```
그때까지만 해도 지훈은 그렇게 확신하고 있었다.
그때까지만 해도 나는 그렇게 확신하고 있었다.
```
→ 다음 씬에서 뒤집힐 것을 암시. 1인칭은 회상형, 3인칭은 자유간접형으로 양쪽 다 사용 가능.

### 2.4 이유 후행 배치
```
지훈은 멈출 생각이 없었다.
아직 이야기하지 못한 게 한 가지 더 남아 있었기 때문이다.
```
→ 결정 먼저, 이유 한 박자 늦게. 긴장 유지.

### 2.7 누적 검수 패턴 (2026-04-25 — 사용자 피드백 30+건 분석 기반)

집필 시점에 미리 점검해 사후 교정을 줄이기 위한 룰. 모든 작품 적용.

#### 2.7.1 호칭·말투 점검
- **어른이 어린아이에게 "자네"·"자넨" 사용 금지** — 어색. 이름·"~야"·"너"·"얘야" 등 자연 호칭으로.
- **인자한 어른은 "~보렴"·"~먹으렴"·"괜찮아"·"쉬어라"** 같은 다정한 평민 톤. 명령형 단답("~해라.", "~있어라.")은 무뚝뚝한 인물에 한정.
- 캐릭터별 말투 시드는 작품 `characters.md`에 명시 → 매 화 첫 등장 시 톤 일치 확인.

#### 2.7.2 능력 시연 = 위장 + 단계 노출
- 주인공이 비범한 능력을 시연할 때, **본인이 의식적으로 위장**해야 자연스러움.
  - "우연이에요" / "그냥 들었더니 그렇게 됐어요" / "신부님이 예전에 한 번 인용하셨던 거 같아서요"
  - 자랑조 응답 ("한 번 읽었어요" 단순 자랑) 금지 — 재수없게 들림.
- **외부 인지는 한 단계 늦춰서 빌드업** — 마을 사람들은 진짜 메커닉(한 번에 외움)을 모르고 "신부의 인용을 들었나 보다" 정도로 정리하는 식. 능력의 정체가 한꺼번에 노출되지 않게.
- 시연 후 보답·반응(빵·고깃덩어리·소문 같은 작은 디테일) 한 줄로 일상에 녹여 정착감 부여.

#### 2.7.3 임팩트 우선 — 약한 표현 점검
- 사건의 임팩트가 약한 묘사는 **구체·미스터리·감각**으로 강화.
  - ❌ "위로 무엇인가 떨어져 자국을 냈다" (밋밋·궁금증 약함)
  - ✓ "새벽에 굉음이 한 번 났는데, 옛 폭약이 터진 게 아니냐는 의심"
- 일화의 임팩트가 약하면 **더 강력한 한 일화로 교체** — 약한 일화 여러 개보다 강한 일화 1~2개.
- 활 일화 패턴: ❌ "끈을 잘라 송아지 옆자리로 떨어뜨렸다" → ✓ "발목 한 뼘 거리의 살무사 머리를 꿰뚫었다"

#### 2.7.4 행동·감각 모순 점검
- 한 단락 안에서 **행동의 시퀀스가 자연스럽게 이어지는지** 자체 점검.
  - ❌ "주머니에 넣었다 / 손에 쥐고 있어야 위로 올라갈 힘이 생길 것 같았기 때문이다." (주머니 vs 손 모순)
  - ✓ "그저 따뜻함이 좋았다 / 카엘은 무심결에 그것을 주머니에 넣었다." (감각 우선 → 행동)
- 매 단락의 동선·자세·감각이 앞 단락과 충돌하지 않는지 확인.

#### 2.7.5 시간 단위·기계적 묘사 회피
- "1번 들었다 / 2번 들었다 / 3번 들었다" 같은 **시간순 기계적 묘사 금지**. 공감·몰입을 깬다.
- 대신 **간결한 동사 한 줄**: "그날부터 카엘은 체력 훈련을 시작했다."
- 변화·성장의 디테일은 다음 화에 결과로 보여주는 편이 임팩트 우선.

#### 2.7.6 시대·문화 단어 점검
- 작품 무대(시대·문화)에 맞지 않는 단어 사용 회피.
  - 판타지 르네상스 무대에 한국식 "기왓장" → "지붕널 조각"
  - 한국 시골 표현이 아닌데 "사흘 길" 같은 공간 표현 → "너무 멀고 험한 길"
- 작품 `worldbuilding.md` §작명/시대 컨벤션 매 화 점검.

#### 2.7.7 어색한·관용 외 표현 점검
- "별칭이 한 바퀴 돈다", "그 정도가 그가 한 일의 전부였다" 같은 **책에서 잘 안 쓰는 표현** 회피.
- "~다는 것을 알았다", "~다는 것을 깨달았다" 둘 중 임팩트가 더 큰 쪽 선택 — 능력·자각의 무게가 클 땐 "깨달았다", 일상 인지엔 "알았다".

#### 2.7.8 러브라인 — 능동·솔직 우선
- 어린 첫사랑·자각 씬은 **애매한 정서 단정으로 잠그지 말고** 능동적 행동·대사·울음 등으로 폭발시켜야 자연스러움.
- 외부 인물의 내면을 서술자가 친절히 풀어주는 식 ("리나는 더 묻지 않는 것이 안전하다는 것을 알고 있었다") 금지.

### 2.6 단어 반복 회피 — 서술 습관 점검 (2026-04-25 사용자 피드백)

특정 단어·표현이 한 화에 4회 이상 반복되면 서술 습관으로 굳어 독자에게 단조롭게 읽힌다. **의식적으로 변주**할 것.

**자주 굳기 쉬운 표현 (점검 대상)**:
- **시간 미세 단위**: "한 박자", "두 박자", "한 박자 늦게" — 한 화 **2~3회 이내**.
  - 대체: "잠깐", "잠시", "한 호흡 늦게", "한 번 늦게", "조금 늦게", "한순간", "0.5초 늦게", "한 음 낮게"
  - 단, 캐릭터의 능력·시그니처 메타포로 의도된 반복은 예외 (예: 카엘의 금빛 "두 박자" 컨트롤)
- **반응 비트**: "한 번 ~했다" — 한 화 5회 이내.
- **호흡 표현**: "한 호흡", "호흡이 길어졌다" — 한 화 3~4회 이내.
- **시선·고개 동작**: "고개를 한 번 끄덕였다", "눈을 한 번 들었다" — 변주 권장.

**점검 방법**: 화 완성 후 `grep -c "박자\|한 호흡\|한 번"` 같은 명령으로 빈도 자체 점검 → 상한 초과 시 변주.

### 2.5 금지 패턴 — "애매한 감정 잠그는 마무리 표현"

POV 외 인물의 내면을 단정 짓거나 씬을 잠그는 다음 식의 마무리는 **금지** (사용자 피드백 2026-04-25, 글투에서 배제):

❌ 나쁜 예 (애매하고 재미 없음):
- "그 정도가 그날 그가 한 일의 전부였다."
- "더 묻지 않는 것이 자기에게 더 안전하다는 것을 그는 알고 있었다."
- "한스는 짧게만 답했다. 길게 말할수록 얽힌다는 것을 그는 잘 알고 있었다."

이유:
1. 외부 인물 내면 직접 서술 (POV 룰 위반).
2. 행동의 동기를 서술자가 친절히 풀어주면서 긴장·여백·캐릭터 신비감을 한꺼번에 죽임.
3. "그가 ~을 알고 있었기 때문이다" 같은 사후 설명은 독자가 이미 추론할 수 있는 것을 작가가 끼어들어 마무리 짓는 패턴.

✓ 좋은 대안:
- 행동·표정·짧은 대사로 끝내고 독자에게 추론을 맡김
- 서사를 능동적 정서·플러팅·행동으로 닫기 (예: 첫 만남이면 부둥켜안고 우는 적극 정서)
- POV 캐릭터 자기 내면은 진입 OK, 외부 인물은 행동만

**1인칭 작품 적용**: D급 용사처럼 1인칭 시점 작품은 모든 게 이미 내면이므로 작은따옴표 없이 그대로 평서문 서술. "그래야 했다.", "이게 마지막일지도 몰랐다." 같이 서술자 톤에 자연 흡수.

**3인칭 작품 적용**: 자유간접화법으로 POV 캐릭터 내면에 깊게 침투. 형식상 "지훈은", "그는"으로 시작해도 내용은 1인칭과 동일한 인지 깊이까지 들어간다. 단, 다른 인물 내면은 절대 직접 서술 X.

## 3. 묘사 = 감각 단문

- 비유·은유 **절제**. 한 줄에 한 감각만.
- 환경 묘사로 씬 시작 금지. **인물 행동·감각**부터.
- 좋은 예: "등까지 들러붙은 뱃가죽에서 꼬르륵거리는 소리가 나고 있었다."
- 나쁜 예: "마치 굶주린 늑대가 뱃속에서 울부짖는 듯한 공복감이 박지훈을 사로잡았다."

## 4. 대사 핑퐁 = 지문 최소

- 4~5턴 연속 큰따옴표만 OK. 누가 누구에게인지는 문맥·말투로.
- 지문은 **전환점 turn에만** 첨부 ("우식이 입술을 떼었다", "그때 우식이 물었다").
- 매 대사마다 "~라고 말했다" 금지.

## 5. 클리프행어 단문 클로징

- 씬 끝·화 끝은 **짧은 한 문장**.
- 예: `"가면 내일이 네 제삿날이 될 테니까."`
- 끝에 길게 정리·요약 금지.

## 6. 씬 전환

- 구분선: `---` (마크다운 표준)
- 시간 점프는 **직설**: "며칠 뒤.", "동네에 있는 포장마차.", "다음 날 오후 2시."
- 전환 후 씬 첫 문장은 **위치·시간·인물 동작** 중 하나로 즉시 진입.

## 6.5 분량 측정 기준 — 리더 카운트 (한국 웹소설 공백 포함 자수)

**측정 도구**: `node scripts/count-chars.mjs <file.md>` 또는 동등한 정규식 기반 카운터.

**카운트 로직** (`src/novel/main.ts:countChapterChars`와 동일):
- 마크다운 헤더 (`# ...`) 제외
- 씬 구분선 (`---`) 제외
- 파라텍스트 (`*N화 끝...*`) 제외
- 이탤릭 마커 `*` 만 제외 (감싼 텍스트는 유지)
- 개행·CR·탭 제외
- **공백·일반 문자 유지** = 한국 웹소설 "공백 포함 자수" 표준

**금지**:
- ❌ macOS `wc -m` 사용 — LC_CTYPE에 따라 바이트 카운트로 동작 (한글 UTF-8 = 3바이트 → 실제 자수의 2.4배 부풀어짐)
- ❌ `wc -c` (바이트 카운트)
- ✅ `node scripts/count-chars.mjs` 또는 `python3` 정규식 카운터

**작품별 가드레일에 분량 명시 시**: 반드시 "리더 카운트 기준" 명기.

## 7. 문장 길이 (느슨한 가이드 — 제한 아님)

문장 길이는 **고정 상한 없음**. 호흡·리듬에 따라 자유롭게.

- **체감 평균 25~40자** 정도가 자연스러움. 그 위·아래 모두 허용.
- 60~80자 긴 문장도 가끔 OK — 절·삽입구가 의미상 한 호흡으로 묶일 때.
- 자르지 말 것: 한 인물의 한 행동·한 인지·한 감각이 자연스럽게 이어지는 흐름은 쉼표·연결어미로 잇는다.
- **단문 강제 구간**: 전투·사망·각성·충격 씬에서만. 일상·내면·관찰 씬은 중문 자유.

**예시 비교**:

❌ 과도한 단문 분할:
```
박씨 할머니의 손가락이 등줄기에 닿았다.
나비는 움직이지 않았다.
털은 평소보다 거칠었다.
척추가 만져졌다.
```

✓ 자연스러운 호흡:
```
박씨 할머니의 손가락이 등줄기에 닿아도 나비는 움직이지 않았고, 털은 평소보다 거칠어서 척추가 그대로 만져졌다.
```

✓ 또는 이렇게:
```
박씨 할머니의 손가락이 등줄기에 닿아도 움직이지 않았다.
털은 평소보다 거칠고, 척추가 그대로 만져졌다.
```

문장을 자르는 기준은 **자수**가 아니라 **호흡·의미 단위**. 한 호흡이면 길어도 한 문장, 두 호흡이면 짧아도 두 문장.
