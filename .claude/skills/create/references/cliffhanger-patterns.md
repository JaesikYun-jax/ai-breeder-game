# 절단신공 패턴 — 6패턴 + 안티패턴 4종

엔딩 클리프행어 카탈로그. episode-architect가 blueprint §7에서 패턴 1개를 명시적으로 선택하고, episode-creator가 해당 패턴에 맞춰 마지막 줄부터 역산한다. quality-verifier가 HOOK 축에서 패턴 일치성과 안티패턴 검출을 확인한다.

> **핵심 원칙**: 위기가 고조되려는 찰나, 절정 부분에서 해결하고 주인공이 끝내주는 보상을 얻으려는 딱 그 순간에 끊는다.

---

## 패턴 1 — 위기 직전 (Pre-Crisis)

### 정의
위기가 터지기 직전에 끊는다. 가장 강력한 절단.

### 예시
```
출구가 보인다.
…한 발만 더 가면 끝.

그때, 등 뒤에서 무언가 묵직한 게 떨어지는 소리가 났다.
```
독자: "뭐가 떨어졌어?!" → 다음 화 클릭.

### 사용 권장 빈도
연재 초반(첫 30화) 4~5화당 1회 / 중반 이후 5~7화당 1회

### blueprint 작성 시 체크
- [ ] 위기의 정체를 마지막 줄에서 **차단**했는가? (다 보여주면 약함)
- [ ] 마지막 줄이 청각/시각/촉각 중 하나의 단편 정보만 제공하는가?

---

## 패턴 2 — 보상 직전 (Pre-Reward)

### 정의
위기를 넘긴 직후, 보상을 막 받기 직전에 끊는다.

### 예시
```
보스가 무너졌다. 진짜로.

상태창에 글자가 떠올랐다.
[클리어 보상: ████████]

뭐?
…뭐라고?
```
독자: "뭐 받았는지 알고 싶다" → 클릭.

### 사용 권장 빈도
사이다 화 직후, 후속 화 시작용으로 잘 어울림

### blueprint 작성 시 체크
- [ ] 보상의 내용을 마지막 줄에서 **차단**했는가?
- [ ] 인물의 첫 반응만 노출하고 보상 자체는 가렸는가?

---

## 패턴 3 — 충격 발견 (Revelation)

### 정의
새로운 정보·진실을 발견하는 순간 끊는다. 떡밥 폭탄.

### 예시
```
스킬 코드를 한 줄 한 줄 따라가다가, 손가락이 멈췄다.

이 함수, 본 적 있는 거야.
…내가 짠 코드잖아 이거.
```
작품 전체 미스터리와 직결된 절단.

### 사용 권장 빈도
중요 떡밥 노출 시. 한 작품에 5~10회.

### blueprint 작성 시 체크
- [ ] 발견의 의미를 화자가 **결론짓지 않았는가**? (P1 위반 위험)
- [ ] 발견의 첫 1차 인지만 노출하고 해석은 다음 화로 넘겼는가?

---

## 패턴 4 — 대사 한 줄 (Dialogue Drop)

### 정의
다른 인물의 의외의 대사 한 줄로 끊는다. 짧고 강력.

### 예시
```
"안강해 씨."

협회장이 책상을 손가락으로 두드렸다. 한 번. 두 번.

"…당신이 누군지, 우리가 알고 있다는 거 아십니까?"
```
인물 등장 임팩트 극대화.

### 사용 권장 빈도
신규 인물 첫 등장 시 자주 활용

### blueprint 작성 시 체크
- [ ] 마지막 줄이 큰따옴표로 끝나는가?
- [ ] 그 대사가 **새 정보·새 의도**를 담고 있는가? (단순 인사·확인은 약함)

---

## 패턴 5 — 시점 차단 (Cutoff)

### 정의
화자가 의식 잃거나 시야 차단되는 순간 끊는다.

### 예시
```
천장이 무너지기 시작했다.
시야가 흔들린다. 머리에 뭔가가 부딪혔다.

마지막으로 본 것은—
```
독자: "그래서 봤다는 게 뭐야!"

### 사용 권장 빈도
드물게. 너무 자주 쓰면 작가 꼼수 티남. 한 작품에 3~5회.

### blueprint 작성 시 체크
- [ ] 차단되기 직전 마지막 정보 조각을 **하나만** 노출했는가?
- [ ] 화자가 **의식적으로 멈춘** 게 아니라 **외부 힘에 의해** 차단됐는가?

---

## 패턴 6 — 시스템 메시지 / 독립 텍스트 블록 (System Cliff)

### 정의
시스템 메시지·문서·코드 블록 등 **독립된 텍스트 덩어리**로 끊는다. 시각적 임팩트 + 떡밥.

### 예시
```
F급 던전 클리어. 보상 처리 중.

…그리고, 그 위로 또 하나가 천천히 나타났다.
나만 보이는, 까만 콘솔.

> ERROR: undefined identifier
> Fallback to legacy mode...
```
이 작품 전용 강력 무기.

### 사용 권장 빈도
1부 4~6회. 핵심 떡밥 노출 시. 작품에 시스템/문서 요소가 있을 때만 사용 가능.

### blueprint 작성 시 체크
- [ ] 텍스트 블록의 내용이 **불완전**한가? (잘리거나 의미가 모호하거나)
- [ ] 본문 마지막 줄이 그 블록 **자체**인가? (블록 뒤에 화자 해석이 붙으면 약함)

---

## 안티패턴 — 절대 금지 4종

다음 패턴은 절단 강도를 즉시 약화시킨다. quality-verifier가 검출 시 HOOK CRITICAL.

### 안티 1 — 풀어진 절단 (Loose Closure)
```
…이렇게 첫 던전이 끝났다. 내일은 또 어떤 일이 있을까.
```
독자: "그래서?" → 다음 화 안 클릭.

**검출 키워드 (Grep)**: 마지막 단락에 다음이 등장하면 의심
- `~이렇게 ~끝났다`, `이렇게 ~지나갔다`
- `내일은 또`, `내일이면`
- `~될 것이다` (마지막 줄 미래 예측)

### 안티 2 — 회상 마무리 (Reflection Coda)
```
나는 한참 동안 그날의 일을 떠올렸다. 그것은 내 인생이 바뀐 순간이었다.
```
P1(결론 선언) + 약한 절단 동시 발동. 최악.

**검출 키워드 (Grep)**: 마지막 단락에 다음이 등장하면 의심
- `~을 떠올렸다`, `회상했다`
- `그것은 ~이었다` (마지막 줄)
- `~한 순간이었다`, `~한 사건이었다`

### 안티 3 — 다음 화 예고 (Spoiler Tease)
```
다음 화에서는 헌터 협회로 향하게 된다.
```
독자를 바보 취급. 절대 금지.

**검출 키워드 (Grep)**:
- `다음 ~에서는`, `다음 화에서`
- `이어진다`, `~할 예정이다`

### 안티 4 — 무의미한 의문문 (Empty Rhetoric)
```
나는 정말로 강해질 수 있을까?
```
추상적 의문. 다음 화 후크 안 됨.

**검출 휴리스틱**: 마지막 줄이 물음표로 끝나면서 다음을 모두 충족하면 의심
- 구체적 대상이 없는 질문 (`정말로`, `과연`, `~수 있을까`)
- 화자 자신에게 묻는 회고적 질문
- 즉각적 사건과 무관한 질문

## 절단 강도 자가 평가 (5문항)

본문 작성 후 다음 항목 ✓:

```
[1] 마지막 줄이 6패턴 중 하나에 해당하는가?
[2] 마지막 줄이 정보를 차단하고 있는가? (다 보여주면 약함)
[3] 마지막 줄이 5단어 이상 30단어 이하인가? (너무 길면 흩어짐)
[4] "[다음 화에 계속]" 빼고 그대로 끝나도 후크인가?
[5] 작가/창작자 본인이 이 절단에서 다음 화 보고 싶은가?
```

5개 중 4개 이상 ✓ → 강도 4 이상.
3개 이하 → 절단 위치 재설계 필요 → REWRITE.

---

## 화별 강도 분배 전략

연속 5화 단위로 강도를 분배한다. 5화 연속 강한 절단은 독자 피로 누적.

| 화 위치 | 권장 강도 | 권장 패턴 |
|--------|---------|----------|
| 5화 묶음 1화째 | 강(4~5) | 패턴 1, 3 |
| 2화째 | 중(3) | 패턴 4, 6 |
| 3화째 | 강(4~5) | 패턴 1, 3 (사이다 직후면 2) |
| 4화째 | 약~중(2~3) | 패턴 2, 6 |
| 5화째 | 최강(5) | 패턴 1, 3 — 다음 아크 입구 |

**유료 전환 직전 화**는 무조건 최강(5) + 패턴 1 또는 3.

## episode-architect blueprint §7 작성 양식

blueprint의 §7 훅 전략 섹션에 다음 필드를 명시:

```markdown
## 7. 훅 전략

### 엔딩 클리프행어
- **선택 패턴**: P{N} ({패턴명})
- **목표 강도**: {1-5}
- **마지막 줄 후보 3개**:
  1. "{후보1 텍스트}"
  2. "{후보2 텍스트}"
  3. "{후보3 텍스트}"
- **차단할 정보**: {위기의 정체 / 보상 내용 / 발견의 의미 / ...}
- **안티패턴 회피 체크**:
  - [ ] 안티 1 (풀어진 절단) 키워드 없음
  - [ ] 안티 2 (회상 마무리) 키워드 없음
  - [ ] 안티 3 (다음 화 예고) 키워드 없음
  - [ ] 안티 4 (무의미 의문) 휴리스틱 없음
```

episode-creator는 이 양식의 마지막 줄 후보 중 하나를 선택하거나 변형하여 사용한다.
