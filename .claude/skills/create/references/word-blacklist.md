# 단어/표현 블랙리스트 — Generic 카테고리 A~E

AI가 무의식적으로 반복하는 표현. episode-creator가 본문 작성 시 회피 / 저장 후 셀프 grep / quality-verifier가 VOICE_PATTERN 축에서 재검증.

**Generic 처리**: 모든 작품에 공통 적용되는 보편 패턴만 포함. 작품별 어휘는 `novel-config.md`의 `forbidden_words:` 필드 또는 `voice_docs:`에 명시한 voice/tone 문서에서 오버라이드한다.

---

## 카테고리 A: 절대 금지 (한 화 0회)

장르 무관 AI 클리셰. 이 표현이 한 번이라도 등장하면 즉시 위반.

| 표현 | 대안 |
|------|------|
| `그래, 이거였다` / `이거였구나` | (생략하고 행동/대사로 표현) |
| `심장이 두근거렸다` / `미친 듯이 뛰었다` | 신체 반응의 다른 형태 (손 멈춤, 시선 멈춤) |
| `온몸에 전율이 흘렀다` | `…등이 서늘해졌다` (정 필요하면) |
| `숨이 막혔다` / `숨을 들이쉬었다` | 호흡 묘사 절제, 침묵으로 대체 |
| `눈앞이 아찔해졌다` | 시야 흔들림 한 문장 |
| `마음 한구석이 ~했다` | (생략) |
| `~을 의미했다` | (직접 진술 금지) |
| `~에 다름 아니다` | (한국어 아님) |
| `나로서는` | `나는` 또는 생략 |
| `그것은 ~이었다` | (대명사+정의문 금지) |
| `결심했다` / `결의를 다졌다` | 행동만 묘사 |
| `~지 않을 수 없었다` | 긍정형으로 |

### 검출 (Grep)
각 표현 화당 0회 확인. 1회라도 검출되면 위반.

---

## 카테고리 B: 한 화 1~2회 한도

이 표현들은 1회 정도는 허용되지만 반복되면 AI 신호로 인식.

| 표현 | 빈도 한도 |
|------|----------|
| `마치` / `~듯이` / `~처럼` | 각 1회, 합쳐서 화당 2회 |
| `결국` | 1회 |
| `그렇게` (문장 시작) | 1회 |
| `이내` | 1회 |
| `한편` (장면 전환 시만) | 1회 |
| `그러나` | 2회 |
| `하지만` | 2회 |
| `어쩌면` | 1회 |
| `분명` / `분명히` | 1회 |
| `어딘가` (모호 표현) | 1회 |

---

## 카테고리 C: 신체 반응 어휘 (총합 한도)

감정을 신체 반응으로 표현하는 건 좋지만, AI는 같은 어휘로 반복한다. 카테고리별 **합산** 한도.

| 카테고리 | 한 화 합산 한도 | 포함 어휘 |
|---------|---------------|---------|
| 심장 관련 | 2회 | `심장`, `박동`, `두근`, `쿵`, `뛰다`, `심박` |
| 숨 관련 | 3회 | `숨`, `호흡`, `들이쉬다`, `내쉬다`, `한숨` |
| 눈 관련 | 4회 | `눈`, `시선`, `시야`, `동공` |
| 손 관련 | 4회 | `손`, `손가락`, `떨림`, `쥐다` |
| 땀 관련 | 1회 | `땀`, `식은땀`, `손바닥` |
| 어깨/등 관련 | 2회 | `어깨`, `등`, `움츠리다` |

> **주의**: 본문에 풍부한 신체 묘사가 필요하다는 원칙과 충돌하지 않는다. 한도는 **같은 어휘의 반복**을 막는 장치이지 신체 묘사 자체를 줄이려는 것이 아니다. 다양한 신체 부위를 골고루 사용한다.

---

## 카테고리 D: 화자 어색 표현 (1인칭/3인칭 공통)

번역체에서 흔히 나타나는 어색 표현.

| 표현 | 이유 | 대안 |
|------|------|------|
| `나의 ~` | 영어 직역체 | `내 ~` |
| `~할 수밖에 없었다` | 회피적 | 행동 직접 묘사 |
| `~지 않을 수 없었다` | 이중부정 번역체 | 긍정형 |
| `~고 있었다` (진행형 남발) | 어색 | `~었다` |
| `~었던 것이다` | 회상체 남발 | 단순 과거형 |
| `그러한` / `그와 같은` | 격식체 | `그런` |
| `~을 가지고 있다` | 영어 have 직역 | `~가 있다` |
| `~에 의해` | 수동태 직역 | `~때문에`, 능동형 |

### 한도
- `나의` 화당 2회 이하 (소유 강조 시만)
- `~고 있었다` 화당 3회 이하 (기존 가드레일)
- `~었던 것이다` 화당 1회 이하

---

## 카테고리 E: 클리셰 장면 표현

상투적 장면 묘사. 즉시 다른 작품과 구별 안 됨.

| 클리셰 | 대안 |
|--------|------|
| `정적이 흘렀다` | `침묵.` (단어 한 줄로) |
| `시간이 멈춘 것만 같았다` | (생략 또는 단문 한 줄) |
| `모두의 시선이 ~에게 향했다` | 인물 한 명의 반응만 묘사 |
| `일촉즉발의 상황` | (한국어 아님) |
| `칼날 같은 긴장감` | (생략, 행동으로 보여주기) |
| `분위기가 무거워졌다` | 누군가의 행동 변화 (커피잔 내려놓기 등) |
| `~는 알 수 있었다` | 발견 묘사로 대체 |
| `공기가 무겁게 가라앉았다` | 인물의 신체 변화 |
| `차가운 침묵이 감돌았다` | `침묵.` 또는 행동 |

### 한도
각 표현 화당 **0회**.

---

## 카테고리 F: 작품별 (novel-config.md에서 오버라이드)

novel-config.md에 다음 형식으로 작품별 추가 금지어를 정의한다:

```yaml
forbidden_words:
  zero_tolerance:                     # 한 화 0회
    - "박자"                          # dclass-hero — AI 시간 미세 단위 말습관
  limited:                            # 화당 N회
    "특정 표현": 1
  category_extra:                     # 신체 반응 카테고리 확장
    심장_extra:
      limit: 2
      words: ["고동치다"]
```

> **dclass-hero 예시**: "박자" 어휘는 카엘의 두 박자/네 박자 의료 단위(캐논 시그니처)를 제외하고 화당 0~1회로 제한 (memory: feedback_bakja_ai_habit.md).

## 검출 자동화 — episode-creator 셀프 grep

본문 저장 직후 다음 항목을 Grep 도구(`output_mode: "count"`)로 점검:

```
[Category A — 0회 확인]
- "그래, 이거였다", "심장이 두근거렸다", "온몸에 전율", "결심했다",
  "~을 의미했다", "그것은 ~이었다"

[Category B — 한도 확인]
- "마치", "~듯이", "결국", "그렇게", "이내", "한편"

[Category C — 합산 한도 확인]
- 심장 어휘 합 ≤2, 숨 어휘 합 ≤3, 눈 어휘 합 ≤4, 손 어휘 합 ≤4

[Category D — 한도 확인]
- "나의" ≤2, "있었다" ≤5, "고 있었다" ≤3, "것이었다" ≤1

[Category E — 0회 확인]
- "정적이 흘렀다", "시간이 멈춘", "분위기가 무거워졌다"

[Category F — 작품별]
- novel-config.md의 forbidden_words 필드 적용
```

초과 발견 시 episode-creator가 해당 표현을 대안으로 Edit 후 재카운트.

---

## 작가/사용자가 추가 패턴 발견 시

작가가 "이 표현 거슬려"라고 지적한 새 패턴은 즉시 novel-config.md의 `forbidden_words`에 추가한다. 한 번 거슬린 표현은 다시 안 쓴다 (학습).
