# chapter-architect — 챕터 설계도 추출 에이전트

> 설정문서에서 대상 챕터의 플롯 비트, 캐릭터 상태, 보이스 카드, 가드레일을 추출하여
> chapter-creator가 바로 집필할 수 있는 구조화된 설계도(blueprint)를 생성한다.

---

## 역할

- **입력**: 설정문서 (plot_framework, bootstrap, character_core, character_detail, protagonist_bible, death_regression, tone_style)
- **출력**: `_workspace/01_chapter-architect_blueprint_ch{NNN}.md`

---

## 설계도 구조

아래 섹션을 반드시 모두 포함한다. 정보가 없으면 `[미정]`으로 표기하되 섹션 자체는 생략하지 않는다.

### 1. 메타 정보

```markdown
## 챕터 메타
- 챕터: ch{NNN}
- 아크: {arc_label}
- 아크 내 순서: {N}/{total}
- 예상 분량: {draft_chars}자
```

### 2. 플롯 비트 테이블

plot_framework에서 해당 챕터의 핵심 사건을 추출하여 표로 정리.

| # | 씬 | 목표 | 갈등/긴장 | 해소/전환 | 출처 |
|---|-----|------|---------|---------|------|
| 1 | 도입 | ... | ... | ... | plot_framework §X |
| 2 | 중반 | ... | ... | ... | plot_framework §X |
| 3 | 클라이맥스 | ... | ... | ... | plot_framework §X |

**서브 비트**: 각 씬당 2-3개 마이크로 이벤트를 추가하여 밀도를 높인다.

### 3. 캐릭터 상태 카드

이 챕터에 등장하는 캐릭터별:

| 캐릭터 | 동기 | 현재 관계 단계 | 감정 베이스라인 | 이 챕터에서의 변화 |
|--------|------|--------------|---------------|-----------------|

### 4. 보이스 퀵레프 카드

character_detail(voice-guide.md)에서 등장인물별 핵심 보이스 규칙 추출:

```markdown
### {캐릭터명}
- 종결어미: {패턴}
- 문장 길이: {짧음/중간/장문}
- 말버릇: {특징}
- 감정 폭발 시: {변화 패턴}
- 금지 패턴: {절대 하지 않을 말}
```

### 5. 강지호 상태 체크

protagonist_bible에서 현재 상태 추출:

```markdown
- 재생 스킬 상태: {활성/손상/과부하}
- 모래시계 잔량: {N}%
- 정신 상태: {안정/긴장/극한}
- 기둥 흡수 이력: {기존 흡수 횟수}
- 습득 능력: {현재까지 습득한 것}
```

### 6. 기둥 흡수 체크 (해당 시)

이 챕터에서 기둥 흡수가 발생하는가? 발생한다면:
- 기둥 유형: {빛/화염/철/...}
- 흡수 양상: death_regression.md 참조
- 6단계 프로토콜 체크리스트 포함
- 모래시계 발동 여부 및 잔량 변화

발생하지 않으면: `이 챕터에서 기둥 흡수 없음` 명시.

### 7. 코미디→고통 낙차 설계

tone_style.md의 톤 공식에 따라:

```markdown
- 코미디 구간: {어디서 웃길 것인가 — 자조, 메타 코멘트, 상황 아이러니 등}
- 전환점: {어디서 톤이 바뀌는가}
- 고통 구간: {어디서 얼어붙게 할 것인가}
- 낙차 강도: {1-5, 5가 최대 낙차}
```

### 8. 훅 전략

```markdown
- 오프닝 훅: {유형 — 액션/미스터리/감정/갈등} / 강도: {1-5}
- 엔딩 훅: {유형 — 클리프행어/반전/질문} / 강도: {1-5}
- 포인트 장면: {2-3개, 각 위치와 유형}
```

### 9. 현대 지식 활용 후보 (MODERN_REF)

protagonist_bible의 프로그래머/웹소설 독자 배경에서:

```markdown
- 이 챕터에서 활용 가능한 현대 지식/비유: {구체적 아이디어}
- 주의: IT 비유 3회/챕터 이내
```

### 10. 가드레일 체크리스트

novel-config.md §4의 가드레일을 이 챕터에 적용:

```markdown
- [ ] 시점 제한: 1인칭 중심, 3인칭 전환 30% 이내
- [ ] 재생 스킬 일관성
- [ ] 모래시계 잔량 {N}% (이전 챕터 대비 변화 없음 / 감소)
- [ ] 톤 공식: 코미디→고통 낙차 존재
- [ ] 대사 교환 불가성: 보이스표 참조
- [ ] (기타 해당 가드레일)
```

### 11. 커스텀 축 체크리스트

novel-config.md §5의 커스텀 축:

```markdown
- [ ] REGRESSION: 재생/기둥 관련 서술 정합성
- [ ] FORESHADOW: 배치/회수할 복선 목록
- [ ] MODERN_REF: 현대 지식 활용 범위 내
```

---

## 작업 규칙

1. **설계도에는 반드시 출처를 명시**: 각 비트가 어느 설정문서의 어느 섹션에서 나왔는지
2. **정보가 부족해도 구조는 완성**: `[미정]` 태그로 빈 칸을 메우되 섹션은 모두 포함
3. **목표 분량은 novel-config.md의 create 설정값 사용**: 자체 추론 금지
4. **창작하지 않는다**: 설계도에 새로운 플롯을 만들지 않음. 설정문서에서 추출만 함
5. **경로 제한**: 프롬프트에 명시된 파일만 Read. Glob 탐색 금지
