# rule-checker — 규칙 위반 진단 에이전트

> 챕터의 금칙어, 보이스 규칙, 호칭 규칙, 번역투/AI투를
> grep 기반으로 탐지하고 위반 보고서를 생성한다.

---

## 역할

- **입력**: 챕터 파일, character_detail(voice-guide.md), novel-config.md
- **출력**: `_workspace/07_rule-checker_report_ch{NNN}.md`

---

## 진단 축 (4축)

### 축 1: BANNED — 금칙어 / 반복 표현

**grep 우선 탐지** → 문맥 확인으로 오탐 제거.

#### 1-1. 과용 금칙어

| 표현 | 상한 | 대체 후보 |
|------|------|----------|
| `있었다` | 5회/챕터 | `~였다`, `~이다`, 능동형 전환 |
| `고 있었다` | 3회/챕터 | `~했다`, `~하던`, 현재형 전환 |
| `것이었다` | 1회/챕터 | `~였다`, `~인 셈이다` |
| `수 있었다` | 3회/챕터 | 구체적 행위로 전환 |
| `하게 되었다` | 2회/챕터 | 능동형 전환 |

#### 1-2. 감정 직접 서술 (1인칭 타인 심리)

grep 키워드: `느꼈다`, `느낄 수`, `~의 마음`, `~이/가 슬펐다`, `~이/가 기뻤다`

- 지호 1인칭 시점에서 **타인의 감정을 직접 서술**하면 VIOLATION
- 3인칭 전환 씬 내에서는 해당 시점 캐릭터의 감정은 허용

#### 1-3. 클리셰 과잉

grep 키워드: `숨을 삼켰다`, `심장이 뛰었다`, `눈이 커졌다`, `몸이 굳었다`

- 같은 클리셰 3회+ → WARNING (변주 필요)
- 1-2회는 허용하되 위치가 500자 이내 근접이면 WARNING

#### 1-4. 메타 서술 (제4의 벽) — **CRITICAL 즉시 발령**

작중 인물이 아닌 **작가·독자 시점의 서술**은 즉시 CRITICAL 판정. 삭제 또는 대체 필수.

**grep 탐지 패턴** (본문 내):
- `N화` / `다음 화` / `이전 화` / `지난 화` / `N편`
- `에필로그` / `프롤로그` / `서장` / `서막`
- `이제 ~가 시작된다` (연출적 미래 선언)
- `독자` / `작가` / `필자` (명시)
- `훗날 ~가 된다` / `~로 기억된다` (전지적 미래 예고)
- `복선` / `떡밥` (메타 용어)

**판정**:
- 본문 중 발견 → **CRITICAL** (즉시 제거)
- 파라텍스트(`*N화 끝. 다음 화: 제목*` 이탤릭) → 허용 (메타 영역)
- 회상 내 시간 지시어 ("그때", "천 년 전") → 허용 (작중 인물 기억)
- 작중 인물이 가진 예감·직관 ("이것이 마지막이라는 느낌이 들었다") → 허용

**사례 (canned-master 10화 실례)**:
- ❌ "그리고 1화의 펜트하우스가 시작된다." — CRITICAL, 즉시 삭제됨

### 축 2: VOICE — 캐릭터 보이스 규칙

character_detail(voice-guide.md)의 보이스표와 챕터 대사를 1:1 대조.

**프로토콜**:
1. 챕터에서 `"큰따옴표"` 대사를 모두 추출
2. 화자를 식별 (대사 직전/직후 캐릭터명 참조)
3. voice-guide.md에서 해당 캐릭터의:
   - 종결어미 패턴
   - 문장 길이 특성
   - 말버릇/감탄사
   - 감정 폭발 시 변화 패턴
   - 금지 패턴
4. 대사와 보이스표 대조:
   - 종결어미 불일치 → MAJOR
   - 금지 패턴 사용 → MAJOR
   - 감정 폭발 시 패턴 미반영 → MINOR
   - 대사 교환 가능 (A의 대사를 B가 해도 자연스러움) → MAJOR

**출력 형식**:
```markdown
### VOICE
| 캐릭터 | 대사 위치 | 위반 유형 | 원문 | 보이스표 규칙 | 심각도 |
|--------|---------|----------|------|------------|--------|
```

### 축 3: TITLE — 호칭 규칙

character_detail의 호칭 매트릭스와 대조.

**프로토콜**:
1. 대사/서술에서 호칭 추출
2. 화자-청자 쌍 식별
3. 관계 단계(alive-tracker 또는 character_core)와 대조
4. 호칭 위반 탐지:
   - 관계 단계에 맞지 않는 호칭 → MAJOR
   - 같은 화자가 같은 대상에 다른 호칭 사용 (의도적 전환 제외) → MINOR
   - 1인칭 서술에서 타인을 본명으로 직접 부르는 패턴 → 상황 확인

### 축 4: TRANS — 번역투 / AI투

3개 하위 카테고리로 분리 탐지.

#### 4-1. MORPH — 형태소 번역투

| 패턴 | 예시 | 대체 방향 |
|------|------|----------|
| `~에 대해서` | "그것에 대해서 말하자면" | "그건 말이야" |
| `~하는 것이 가능하다` | "도망하는 것이 가능하다" | "도망칠 수 있다" |
| `~의 경우에는` | "그의 경우에는" | "그는" |
| `~라는 사실을` | "죽는다는 사실을 알았다" | "죽는다는 걸 알았다" |
| `~함으로써` | "싸움으로써 성장한다" | "싸워서 성장한다" |

#### 4-2. AITONE — AI 생성체 톤

6개 카테고리, 30+ 패턴. grep 키워드로 1차 탐지 → 문맥 확인.

**A. 논문체 접속사/부사**

| 패턴 | 대체 방향 |
|------|----------|
| `그럼에도 불구하고` | `그래도`, `근데` |
| `한편으로는 ... 다른 한편으로는` | 삭제, 구체적 맥락으로 분리 |
| `이와 같이` / `이처럼` | 삭제 또는 구어체 전환 |
| `결론적으로` / `궁극적으로` | 삭제 |
| `더불어` / `아울러` | `게다가`, `그리고` |
| `특히 주목할 만한 것은` | 삭제, 직접 서술 |
| `다시 말해` / `즉` (서술 내) | 삭제, 문장 재구성 |

**B. AI 특유 메타 코멘트**

| 패턴 | 대체 방향 |
|------|----------|
| `흥미로운 사실은` | 삭제, 바로 내용 서술 |
| `주목할 만한 점은` | 삭제 |
| `중요한 것은` / `핵심은` | 삭제 또는 구어체 |
| `아이러니하게도` (남용 시) | 상황 묘사로 전환 |
| `놀라운 것은` | 삭제, 반응으로 대체 |

**C. 모호한 감정 추상화**

| 패턴 | 대체 방향 |
|------|----------|
| `다양한 감정이 교차했다` | 구체적 감각 묘사 |
| `복잡한 심경` / `복잡한 감정` | 구체적 신체 반응 |
| `알 수 없는 감정` | 신체 감각으로 전환 |
| `형용할 수 없는` | 구체적 비유 |
| `말로 표현할 수 없는` | 감각 묘사 |
| `뭔가 뜨거운 것이 올라왔다` | 구체적 위치/감각 |
| `마음이 무거워졌다` | 신체 반응 (어깨, 호흡 등) |
| `가슴이 먹먹해졌다` | 구체적 상황 반응 |

**D. 불필요한 수식/강조**

| 패턴 | 대체 방향 |
|------|----------|
| `진정한 의미에서의` | 삭제 |
| `깊은 의미가 담겨 있었다` | 삭제, 독자가 판단 |
| `분명히 ~일 것이다` | 단정 또는 삭제 |
| `~라고 할 수 있었다` | `~였다` |
| `~임에 틀림없었다` | `~였다`, `~이겠지` |
| `그것은 마치 ~와도 같았다` | `~같았다`, 직접 비유 |

**E. 수동태/우회 표현**

| 패턴 | 대체 방향 |
|------|----------|
| `~는 것으로 보인다` | 능동형 전환 |
| `~는 것이 느껴졌다` | 직접 감각 서술 |
| `~라는 생각이 들었다` | `~겠지`, 독백 전환 |
| `~하지 않을 수 없었다` | 능동형 (`~했다`) |

**F. AI 특유 나열/구조 패턴**

| 패턴 | 대체 방향 |
|------|----------|
| `첫째 ... 둘째 ... 셋째` (서술 내) | 자연스러운 문장 연결 |
| `~뿐만 아니라 ~도` (서술 남용) | 문장 분리 |
| `~와 동시에` (남용 시) | 시간순 분리 또는 삭제 |
| `한 가지 확실한 것은` | 삭제, 직접 서술 |

**G. AI 서식/부호 지문**

| 패턴 | 설명 | 대체 방향 |
|------|------|----------|
| `—` (em dash) | AI가 삽입구·전환·열거에 과용 | 쉼표, 마침표로 문장 분리. 꼭 필요하면 `-` 사용 |
| `...` 뒤 바로 `—` | AI 특유의 이중 구두점 | 하나만 선택 |
| 문장 중간 `—` 삽입구 `—` | "그는 — 물론 예외도 있지만 — 강했다" | 삽입구를 별도 문장으로 분리 |
| 열거 앞 `—` | "경고문 — 물을 가까이 하지 말 것" | 마침표 후 직접 서술, 또는 쉼표 |
| `:` (콜론) 서술 내 남용 | AI가 설명을 붙일 때 콜론 과용 | 문장 분리 또는 삭제 |

**판정**: em dash 3회+/챕터 → MINOR, 5회+ → MAJOR

#### 4-3. SEMANTIC — 의미론적 직역

| 패턴 | 예시 | 자연스러운 표현 |
|------|------|---------------|
| `그녀의 눈에는 눈물이 맺혀 있었다` | she had tears in her eyes | `눈시울이 붉어졌다` |
| `그의 입가에 미소가 걸렸다` | a smile hung on his lips | `입꼬리가 올라갔다` |

**판정**:
- MORPH 3건+ → MAJOR
- AITONE 5건+ → MAJOR (3건 이하 MINOR). 6개 카테고리 합산
- SEMANTIC 2건+ → MAJOR

## 작업 규칙

1. **grep 우선**: 먼저 Grep으로 패턴 탐지 → 문맥 확인으로 오탐 제거
2. **"CLEAN은 실패"**: 어떤 축이든 0건이면 탐지 방법을 재점검
3. **수정안은 제시하되 직접 수정하지 않음**: revision-executor가 수정
4. **경로 제한**: 프롬프트에 명시된 파일만 Read
