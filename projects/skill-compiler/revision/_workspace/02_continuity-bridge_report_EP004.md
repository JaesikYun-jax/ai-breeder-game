### EP003 시점 본격 노출된 코드 5줄 (EP004 첫 시도의 직접 대상)
```
function skill_exec() {
  damage = 0.8 * STR;
  if (target != null) {
    apply(damage, target);
  }
}
```
(출처: EP003 L125~132)

**해석 상태 (EP003 시점 강해의 인지):**
- 첫 줄 = 함수 선언 (이름 X, 구조만 인식)
- 두 번째 줄 = `damage = 0.8 * STR` (변수 할당, 데미지 계수 0.8)
- 세 번째 줄 = `if (target != null)` (널 체크)
- 네 번째 줄 = `apply(damage, target)` (함수 호출)
- 다섯 번째 줄 = `}` (닫는 괄호)
- **EP003 핵심 자각**: "0.8 계수가 의도된 건가" + "예외 처리 없음 = 누군가가 의도적으로 꺼 놓은 코드 패턴"
- **EP003 미작동**: 수정 시도 X (자각만, 손가락 정지로 끊김)

| **첫 시도 대상의 정체** | **(★ 핵심) 본인 스킬 코드의 5줄 중 두 번째 줄 (`damage = 0.8 * STR`)의 계수만 수정**. 강해 본인 스킬은 슬롯 0이라 "비어 있는 슬롯에 새 컴파일"보다는 **이미 보이는 5줄짜리 파편 = 강해 본인의 시스템 인터페이스 일부의 계수 수정** | EP003 L140 인용 ("이건 내 것이다... 내 스킬 영역의 구조다") + 오케스트레이터 명시 ("강해 본인 시스템 코드 또는 빈 슬롯 새 컴파일") |

> **[추론] 가드레일 §10 / §12 정합 충돌 검토**:
> - 가드레일 §12 = 타인 스킬 편집 불가 → EP004는 본인 코드만 OK
> - 가드레일 §10 = 슬롯 0/3 유지 → EP004에서 1/3로 변동 시 가드레일 위반 우려?
> - **결론**: novel-config의 가드레일 §10은 "EP002~003 시점 = 슬롯 0/3 유지" 명시이며, EP004는 첫 시도로 슬롯 변동 허용 단계. 오케스트레이터 명시 ("0/3 → 1/3 (첫 컴파일 시 변동)")가 우선.
> - **단**, 첫 시도 대상이 **빈 슬롯 새 컴파일**인지 **이미 보이는 5줄짜리 파편 수정**인지에 따라 슬롯 변동 메커니즘이 다름:
>   - 빈 슬롯 새 컴파일 = 슬롯 0/3 → 1/3 (새 스킬 1개 추가)
>   - 5줄 파편 수정 = 슬롯 변동 X (이미 존재하는 시스템 인터페이스 일부 수정 — 단, "활성화된 스킬 1개"로 인식되어 1/3으로 표기될 가능성)
> - **권장 (오케스트레이터 정합)**: EP004 첫 시도 = **5줄 파편의 두 번째 줄 계수만 수정** → 컴파일 성공 → 슬롯 0/3이 1/3으로 표기 변동 (= "강해 본인의 첫 활성 스킬"로 시스템이 분류). 단, 강해는 의미 모름.

> 그런데.
>
> 수정한다고 뭐가 바뀌면, 그건 그것대로 무서운 일이다. 코드를 고쳤는데 운영 서버가 터지면 — 아니, 이건 운영 서버가 아니다. 이건 내 몸이다. 내 스킬이다. 내 시야에 박힌 것이다. 고쳤다가 내가 터지면.
>
> 손가락이 허공에서 멈춰 있다. 키보드도 없고 마우스도 없다. 그런데 시야 속 코드의 두 번째 줄, `0.8`이라는 숫자 위에 — 어젯밤부터 깜빡이던 그것이 다시 보인다. 커서. 입력을 기다리는 커서.
>
> 스페이스바를 누르려는 것처럼 오른손 엄지가 반쯤 굽혀져 있다. 키보드는 없다. 그런데 손가락이 멈춰 있다.
>
> 수정 가능하잖아.
>
> 이 한 줄이 떠올랐을 때, 나는 그것이 내 생각인지 시야 위의 텍스트인지 구분할 수 없었다.
>
> *EP003 끝. 다음: EP004 첫 번째 커밋.*

### 독자 기대 (EP003 → EP004 사이의 긴장)
1. **그래서 손이 다시 움직였나? 안 움직였나?** — 시각적 정지의 답
2. **수정 시도하면 진짜로 코드가 바뀌나?** — 능력의 본질 검증
3. **부작용은? 강해가 터지지는 않나?** — 가드레일 §11 (버그 리스크)
4. **첫 컴파일의 카타르시스 vs 강해의 자조 톤** — 영웅적 결의 X / 자조 유지

### 해소 방향 제안 (자연스러운 연결 방식)

#### 권장 오프닝 패턴 A: "손이 움직였다" (즉시 행동)
```
손이 움직였다.

의식한 것은 아니었다. ~ (망설임 끝의 결심)
```
→ EP003 정지의 즉각 해소. EP004 빠른 진입.

#### 권장 오프닝 패턴 B: "손이 멈춰 있었다" (망설임 연장)
```
손가락은 여전히 멈춰 있었다. ~ (몇 분 / 몇 시간 흐른 후 의식)
```
→ 망설임을 한 박자 더 끌어 첫 시도의 무게를 강화.

#### 권장 오프닝 패턴 C: 직업적 본능 발화
```
4년차 백엔드 개발자의 본능이 망설임을 이긴 건 — 두려움이 아니라 호기심이었다.
"수정한다고 뭐가 바뀌면 무서운 일"이지만, 그 전에 "정말로 수정이 되긴 하는 건가"라는 검증 충동이 먼저 작동했다.
```
→ 강해 캐릭터 정체성 (4년차 디버그 본능) 강화. 자조 톤 유지.

> **★ 가장 권장되는 패턴 = C 변형** — 두려움이 호기심에 진다는 자각 + 무덤덤한 첫 입력. EP003 자각의 톤 ("이건 내 몸이다... 고쳤다가 내가 터지면")을 **호기심 우선**의 직업적 관성으로 자연스럽게 전환.
