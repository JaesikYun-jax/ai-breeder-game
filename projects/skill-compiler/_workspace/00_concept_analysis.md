# 컨셉 분석 — 나에게만 스킬이 코드로 보인다

> Phase 1 결과 (design-big 2026-05-09)

---

## 작품가제

**스킬컴파일러**

## 전문 분야

- 백엔드 개발 (중소 SI 4년차 프로그래머)
- 현대 한국 헌터 시스템 (대각성 1년 후)

## 서사 기점

- **시대**: 현대 한국, 전 인류 동시 각성 '대각성(Great Awakening)' 1년 후
- **계기**: F급 비각성자 안강해가 자신의 스킬이 수정 가능한 코드로 보인다는 것을 자각 (EP003)
- **배경**: 10년 전부터 혼자 상태창을 보았으나 정신병원 이력 → 사회적 고립 → 코드 속으로 도피

## 핵심 차별화 포인트

| 흔한 헌터물 | 이 작품 |
|---|---|
| 회귀로 미래 앎 | 능력으로 평생 외톨이였던 정체성 |
| 시스템은 신비로움 | 시스템은 코드, 분석 가능 |
| 강한 스킬을 받음 | 약한 스킬 조합으로 강해짐 |
| 직업 = 단순 설정 | 직업 = 트라우마의 결과물 |

## 타겟 플랫폼

**문피아** (canonical)

## 톤 & 무드

- 1인칭 자조/냉소
- 사이다 (무시→처발림→충격 사이클)
- 코드 시각화 (마크다운 코드 블록)
- 미스터리 (시스템 농장 떡밥 — 절제)
- 톤 공식: 사이다 4 : 자조 4 : 미스터리 2

## 유사 작품군

1. **나 혼자만 레벨업** — 한국 헌터물 레퍼런스, 솔플 + 성장 + 시스템. 차이: 회귀 X, 코드 편집 메커닉
2. **전지적 독자 시점** — 시스템 해체 + 메타적 관점. 차이: 프로그래머 직업 리얼리즘, 전투 방식(코드 수정)
3. **소드 아트 온라인** — 가상/시스템 세계 속 생존. 차이: 현실 한국 배경, 코드 직접 조작

## 예상 분량

- 1부: 약 80화 (5,000~7,000자/화)
- 전체: 200~300화 (3부 구성)

## 리서치 필요 카테고리

- R3: 한국 IT 업계/백엔드 개발자 구조, 한국 웹소설 헌터물 조직 구조
- R4: 한국 IT 산업 사건 연표 (서사 배경 소재), 헌터물 장르 트렌드
- R5: 기존 헌터물/시스템물 분석 (나혼렙, 전독시, 템빨 등)
- R6: IT 업계 갈등 사례 (빌런 모티프), 헌터물 길드 갈등 패턴
