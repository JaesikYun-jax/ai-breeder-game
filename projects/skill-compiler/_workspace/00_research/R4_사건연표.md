# R4: 사건 연표 -- 한국 IT/게임 산업 + 웹소설 헌터물 트렌드 (2015~2026)

**신뢰도**: MEDIUM
**근거**: 웹 검색 불가로 학습 데이터(~2025.05) 기반. IT 산업 사건은 주요 보도 기반, 웹소설 트렌드는 플랫폼 공개 데이터와 업계 분석 기반. 미확인 항목 약 20%.

## 3. 문피아 플랫폼 트렌드

### 3-1. 문피아 핵심 특성

| 항목 | 내용 |
|------|------|
| **핵심 독자층** | 20~30대 남성 (70%+), IT/게임 친화적 |
| **장르 강점** | 현대 판타지, 헌터물, 먼치킨, 회귀물 |
| **유료 전환** | 보통 30~50화 무료 후 유료 전환 |
| **연재 주기** | 일 1화 또는 주 5~7회가 경쟁력 있는 패턴 |
| **분량** | 회당 4,000~6,000자 (문피아 표준) |
| **노출 알고리즘** | 최신 업로드 시간, 추천수, 조회수, 연재 지속성 반영 |

### 3-2. 문피아 성공 패턴

| 요소 | 성공 패턴 |
|------|----------|
| **첫 5화** | 세계관 설명 최소화, 즉시 사건/능력 시연 |
| **10~20화** | 먼치킨 쾌감 확립, 첫 대형 사이다 |
| **30~50화** | 유료 전환 직전 대형 클리프행어 |
| **연재 안정성** | 휴재 최소화, 일정 유지가 핵심 |
| **독자 소통** | 작가 후기, 댓글 반영, 팬 커뮤니티 |

## 4. 서사 활용 제안

### 4-1. 시대 배경 설정
- 배경: "대각성"이 일어난 지 1년 후 (현실 기준 2025~2026년으로 설정 시)
- AI 시대에 개발자 위기감을 느끼던 안강해가, 각성이라는 새로운 변수를 만남
- IT 업계의 구조조정 + 각성자 사회의 등장이 겹치는 혼란기

### 4-2. 장르 트렌드 활용
- 나혼렙 이후 8년, 독자들은 F급 성장물에 내성이 있음. 차별화 필수.
- "스킬 코드 편집"은 IT/코딩 소재 헌터물 중 대형 히트작이 없는 빈 공간
- 시스템 해부(시스템 = 의념 수확 농장) 테마는 2024~2026 트렌드와 정확히 맞물림
- 1인칭 자조 화자는 문피아 독자층(20~30대 IT 친화 남성)에 최적화

### 4-3. IT 사건의 서사 전환
- 카카오 데이터센터 화재 → 던전 관리 시스템 대규모 장애, 미등록 던전 폭주
- AI 코딩 도구 → 스킬 코드를 자동 수정하는 "오토 컴파일러" 아이템 (경쟁자 등장)
- 개발자 구조조정 → 헌터 사회에서도 저등급 헌터 구조조정 (F급 퇴출 정책)
- 해킹 사건 → 시스템 해킹, 각성자 데이터 유출, 등급 조작 스캔들
