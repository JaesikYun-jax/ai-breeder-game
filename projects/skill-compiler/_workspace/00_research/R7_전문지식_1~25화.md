## 1. SI 4년차 백엔드 개발자 일상

### 1-1. 출퇴근 동선과 공간

**원청 빌딩 출입 풍경 (강남/판교)**:
- 1층 로비 회전문 → 안내데스크 → 보안 게이트(스피드게이트). 정직원은 사원증을 태그하면 게이트가 부드럽게 열린다. **협력사는 별도의 임시출입증 또는 색깔이 다른 발주처 발급증**(보통 빨강/주황 띠 또는 "협력사" 글씨 인쇄)을 패용. 정직원 사원증은 회사 로고 + 본인 사진 + 풀네임. 협력사증은 "외주", "협력사", "PARTNER" 표기, 사진 없이 이름만. 같은 게이트를 통과하지만 디자인 자체가 갈린다.
- 엘리베이터: 같이 탔는데 정직원 끼리 이야기하면 협력사는 입을 닫는다. 층 버튼을 누르는 손에 미세한 머뭇거림.
- 출입증을 깜빡 두고 오면 1층에서 협력사 운영팀에 전화 → 임시증 발급 → 30분~1시간 대기. "본인 회사가 아니라 발주처에 손 빌리는 절차".

**자취방 → 사무실 통근**:
- 강해 같은 비주류 4년차의 전형적 자취 위치: 신도림·구로디지털단지·문래·합정·신림·역삼·서초 변두리 (월세 50~70만 원, 보증금 1~3천만 원, 5~7평 원룸).
- 9호선 급행 / 2호선 / 신분당선이 IT 통근의 3대 동맥. 강남역·삼성역·판교역 출구에서 정장이 아닌 후드/맨투맨/슬랙스+스니커즈 무리가 우르르 쏟아진다. 사원증을 목에 건 사람들이 줄지어 빌딩 로비로 빨려 들어감.
- 오전 8:50 무렵 1층 카페에서 아메리카노 한 잔(2,500~3,500원). 정직원은 사내 카페에서 무료/할인, 협력사는 1층 외부 매장 이용.

**자취방 풍경 (강해 묘사용 디테일)**:
- 27인치 듀얼 모니터(중고로 산 LG 27UL550 같은 모델), 기계식 키보드(레오폴드 FC660M 또는 키크론), 게이밍 의자가 아니라 시디즈 T50 정도. 책상 한 켠에는 비타500 빈 병, 박카스, 컵라면 용기 정리 안 됨.
- 옷장에 후드집업 4벌 같은 색. 검정·차콜·네이비.

### 1-3. 갑질 패턴 (강해 일상에 녹이는 미세 디테일)

**거시적 갑질**:
- **요구사항 변경**: 스프린트 시작 후 PM이 "고객사 요청이 바뀌어서요" → 이미 배포된 기능을 갈아엎음. 책임은 개발자에게.
- **일정 압박**: "다음 주 월요일까지 가능하시죠?" (금요일 17시 통보).
- **기술 의사결정 배제**: 아키텍처 회의에 협력사는 안 부름. 결정된 후 "이렇게 가기로 했으니 구현 부탁드려요".
- **성과 귀속**: 리팩토링으로 응답속도 80% 단축 → 발표는 정직원 시니어가. 협력사는 슬라이드 한 줄 ("외주 협력 감사").

**미시적 갑질** (강해 자조의 핵심 소재):
- **식사 차별**: 정직원 사내식당 입장 카드 안 됨. 같은 층 회의 끝나도 "협력사분들은 따로 점심 어떻게 하세요?" 미세한 분리.
- **화장실 분리**: 같은 층 다른 화장실. 명목상 "보안 구역" 때문이지만 실제로 정직원과 동선이 겹치지 않도록.
- **휴게실 분리**: 정직원 라운지에 협력사 출입 불가. "보안 회의 자료가 있어서요".
- **회의실 자리 배치**: 발주처가 상석, 협력사는 입구 가까운 끝자리. 빔프로젝터를 등지고 앉음.
- **회식 강제**: "이번 분기 회식 협력사분들도 참여하시죠" → 안 가면 다음 주 분위기 어색.
- **호칭의 미세함**: 정직원은 "OO님" 풀네임, 협력사는 종종 "거기 OO씨" 또는 "협력사분".
- **회식 자리**: 정직원 시니어 옆자리는 정직원 주니어. 협력사는 항상 끝자리.
- **사원증 색깔**: 같은 빌딩 안에서도 정직원은 흰색, 협력사는 빨간 띠. 한 눈에 구분.
- **음료 대접**: 회의실 음료가 정직원 회의에는 스타벅스, 협력사 참여 회의에는 회의실 정수기 종이컵.

### 1-4. 4년차의 직무 이해도 (강해의 능력 근거)

- **혼자 기능 단위 개발 가능**: API 엔드포인트 하나, 배치 잡 하나 정도는 설계부터 배포까지 단독 수행.
- **아키텍처 결정권 X**: "이 도메인은 어떤 패턴으로 갈까요" 회의에 부르지 않음. 결정 통보만 받음.
- **레거시 코드 분석 능숙**: 4년간 남이 쓴 코드 디버깅을 가장 많이 했음. 로그 trace, git blame, IDE의 Find Usages를 본능적으로 사용. = **"찾아내는 눈"** = 작품에서 강해의 핵심 자질.
- **프레임워크/라이브러리 표면적 이해**: Spring 어노테이션은 다 쓸 줄 알지만, 내부 구현(IoC 컨테이너 라이프사이클)까지는 모름. **"동작은 시키는데 왜 그런지는 가끔 모름"** = 강해가 시스템(상태창) 앞에서 느끼는 감각과 같음.
- **테스트 코드 작성**: JUnit5 + Mockito 정도. 통합 테스트는 잘 안 씀.
- **DB 튜닝 초급**: EXPLAIN을 보긴 하지만, 옵티마이저 통계 같은 건 모름. 인덱스 추가는 시니어 결재.
- **AWS 콘솔 익숙**: EC2 SSH, RDS 접속, S3 업로드, CloudWatch 로그 검색. IAM 정책 수정은 권한 없음.

## 2. Java/Spring/MySQL/AWS 기술 디테일 + 코드 시각화 예시

### 2-1. 변수/메서드 시그니처 기본

**Java 17 기준 (강해 회사가 LTS 채택)**:

```java
@RestController
@RequestMapping("/api/v1/users")
@RequiredArgsConstructor
public class UserController {
    private final UserService userService;

    @GetMapping("/{id}")
    public ResponseEntity<UserResponse> getUser(@PathVariable Long id) {
        return ResponseEntity.ok(userService.findById(id));
    }
}
```

**자주 쓰는 어노테이션**:
- `@RestController`, `@Service`, `@Repository`, `@Component`
- `@Transactional`, `@Async`, `@Scheduled`
- `@Valid`, `@NotNull`, `@Size`
- `@RequestMapping`, `@GetMapping`, `@PostMapping`, `@PathVariable`, `@RequestBody`
- Lombok: `@Getter`, `@Builder`, `@RequiredArgsConstructor`, `@Slf4j`

### 2-2. MySQL 디테일

**EXPLAIN 결과 해석**:
```
| id | select_type | table | type  | key      | rows | Extra       |
|  1 | SIMPLE      | users | const | PRIMARY  |    1 | Using index |
```
- `type`: const < eq_ref < ref < range < index < ALL (오른쪽으로 갈수록 나쁨)
- `key`: 사용된 인덱스. NULL이면 풀스캔.
- `rows`: 스캔할 예상 행수.
- `Extra`: "Using filesort", "Using temporary"가 보이면 위험.

**전형적 슬로우 쿼리 패턴**:
```sql
SELECT * FROM orders WHERE DATE(created_at) = '2026-05-09';
-- 함수가 인덱스를 무력화. 4년차 강해는 본능적으로 찾아낸다.
```

### 2-3. AWS 기초 (강해 사용 범위)

- **EC2**: t3.medium, t3.large 정도 인스턴스. SSH 접속 키페어 관리.

- **S3**: 파일 업로드/다운로드. presigned URL.
- **CloudWatch**: 로그 그룹에서 검색(필터 패턴). 알람은 시니어가 설정.

#### 예시 8 -- 디버깅 미니 시퀀스 (인라인 사용)

- `damage = 0.8 * STR` 한 줄을 보고 "STR 가중치가 0.8이라니, 누가 이렇게 짜놨지" 자조.
- `if (target.armor < 0.3)` 조건문이 빠진 걸 보고 "이래서 약체에게도 안 박힌다" 진단.
- `cooldown = 30s`을 보고 "쿨이 30초인 D급이 어떻게 1군에 있어" 한탄.

### 2-7. 리팩토링 패턴 (EP008~ 핵심 메커니즘)

- **Extract Method**: 중복 로직을 메서드로 추출.
- **Rename Variable**: 의미 불명확한 변수명 개선.
- **Inline Variable**: 한 번만 쓰는 변수 인라인.
- **Extract Interface**: 결합도 낮추기.
- **Replace Conditional with Polymorphism**: if-else 사슬을 다형성으로.
- 강해의 작품 내 응용: "이 스킬의 분기 5개를 폴리모피즘으로 짜면 한 줄에 통합되는데" → 등급 그대로 효율 폭증.

## 3. 헌터 사회 디테일 (EP001~025 활동 무대)

### 3-1. 한국 헌터 협회 등급 측정실

- **위치**: 강남 협회 본부 지하 1층(가상 설정). 던전 코어 일부를 인공적으로 매립한 "측정 챔버".
- **절차**:
  1. 접수 → 서류 작성 → 동의서 (사망/부상 면책)
  2. 측정복 환복 (회색 무지 트레이닝)
  3. 측정 챔버 입장 → 코어 잔류 마나 노출 → 상태창 측정
  4. 결과 출력 → 공식 등급증 발급
- **F급 판정 풍경**: "측정 불가" 메시지가 LED 패널에 빨갛게. 직원의 표정: 안쓰러움 80% + 호기심 20%. "...다시 측정해볼까요?" (이미 3번째). 강해의 자조: "오늘도 측정 불가. 인생 패턴이다."

### 3-2. 던전 출입 절차

| 단계 | 내용 |
|---|---|
| 게이트 등급 측정 | 협회 측정관이 출현 즉시 파견. F급/E급/D급/C급/B급/A급/S급. |
| 공략 등급 제한 | E급 게이트는 D급 이상 헌터 출입 권장. F급 단독 진입은 자살 행위로 분류, 실제로는 출입 가능(법적 제한 X) |
| 보험 가입 | 헌터 보험. F급은 가입 거부되거나 보험료 10배 |
| 출입증 태그 | 게이트 입구 협회 부스에서 출입증 태그 → 들어간 시간/등급/소지품 기록 |
| 입장 | 게이트 통과. 비각성자도 물리적으로는 들어갈 수 있음 (위험 자기 책임) |
| 클리어 보고 | 코어 처치 후 게이트 자동 닫힘. 협회에 드롭 아이템 등록(과세 대상) |

### 3-3. 무능력자 부서(언랭커과)

- **공식 명칭**: 한국 헌터 협회 무능력자관리과 (통칭 언랭커과).
- **소속**: 협회 본부 6층 한 켠.
- **인력**: 약 30명. 행정 25명 + F급 비전투 5명.
- **업무**: F급 등록·관리, 던전 잔해 청소, 행정 보조, 사망자 유품 처리, F급 통계 작성.
- **분위기**: 햇빛 안 드는 사무실. 형광등. 중고 의자. 책상마다 박스 더미.
- **최서진(F급, 행정 담당)이 일하는 공간**: 모니터 2개, 한쪽엔 협회 ERP, 한쪽엔 카카오톡 PC. 강해가 처음 출근하면 "어, 그 유일한 F급 분이시구나" 미세한 호기심.

### 3-4. 5대 길드 사옥/분위기

- **헥사 (HEXA)**: 강남 신논현역 인근, 30층 자체 빌딩. 1층 로비에 길드 마크(육각형 + 검) 대형 조형물. 정직원 양복, 보안요원 검은 정장 + 이어피스. 출입 게이트 3중. 강해 같은 F급은 1층 안내데스크에서 막힘.
- **솔라리스 (Solaris)**: 판교 위워크 임대 5개 층. 캐주얼 분위기, 후드/맨투맨이 다수. 카페 같은 라운지. 윤세하가 직접 1층까지 내려와 인사하는 스타일.
- **뱅가드 (Vanguard)**: 광화문 정부청사 옆 별도 동. 공무원 분위기. 사복도 단정한 비즈니스 캐주얼. 회의 시작 정시.
- **크로스보더 (Crossborder)**: 여의도 IFC. 영어 표지판 병기. 보안 4중. 사내 카페가 외국계 체인.
- **넥스트 (NEXT)**: 성수동 리모델링 사옥. 컨테이너 + 노출 콘크리트. 평균 연령 28세. 자유로운 출퇴근. 강민준이 직접 후드티 차림으로 면접.

### 3-5. 헌터 갤러리/SNS/길드 인플루언서

- **헌갤** (헌터 갤러리, 디시인사이드 가상): F급 비방, 솔플 영상 클립, 길드 내부 폭로, 등급 측정 후기. "F급 유일 사례" 강해는 EP020 이후 헌갤에서 화제 → 익명 조롱(사이다 삼키기 전 단계).
- **유튜브 헌터 채널**: 길드 마스터들이 직접 운영. 진현승은 안 함(품격), 윤세하는 함(브랜딩), 강민준은 라이브 자주 함(스타트업 마인드).
- **인스타그램**: A급 헌터들의 인스타. 던전 셀카, 드롭 아이템 자랑. 한지유는 인스타 비공개.
- **트위터**: 협회 비판, F급 인권 운동, 던전 사고 폭로. 익명 계정 다수.

### 3-6. F/E/D급 던전 풍경

| 등급 | 모습 | 몬스터 | 보상 |
|---|---|---|---|
| F급 | 도심 골목, 폐건물, 지하상가 / 어두운 안개 / 1~2층 규모 | 슬라임, 고블린 새끼, 대형 쥐 | 50만~200만 원 잡템 |
| E급 | 폐공장, 폐지하철, 산기슭 / 약한 마나 흐름 / 3~5개 방 | 고블린, 코볼트, 늑대 | 200만~1,000만 원 |
| D급 | 폐쇄 학교, 폐병원, 동굴 / 명확한 마나 / 다층 구조 | 오우거 새끼, 워그, 그림자 거미 | 1,000만~5,000만 원 |

### 3-7. 솔플 vs 파티 절차 차이

**솔플**:
- 협회에 단독 진입 신고 → 보험 미가입 시 면책 동의서 → 출입증 솔로 모드 태그.
- 응급 통신기(Emergency Beacon) 휴대 의무.
- 사망 시 유품 회수만 협회 보장. 시신 회수는 D급 이상부터.

**파티**:
- 파티 등록 → 리더 지정 → 보상 분배율 사전 등록.
- 협회 분쟁 조정실에서 분배 갈등 중재.
- F급은 "파티원 구색"으로도 자주 거부됨.

## 4. 코드 시각화 묘사 디테일

### 4-1. 강해의 시야에 코드가 떠오르는 방식

**기본 묘사 원칙**:
- **반투명 텍스트** + **시야 전체에 오버레이**. 다른 사람 눈에는 안 보임.
- **모노스페이스 폰트** (강해 머릿속에서 익숙한 D2Coding 또는 SF Mono 같은 느낌)을 환각처럼 인식.
- **녹색** = 활성/정상. **노란색** = 경고. **빨간색** = 에러. **회색** = 비활성/주석. **흰색** = 일반 코드.
- 손짓이 아니라 **시선**으로 커서 이동. 익숙해지면 무의식.
- 빌드 룸이 아닌 일상에서는 **코드가 흐릿하게 흘러가는 배경**처럼만 보임. 집중하면 또렷해짐.

**대표 묘사**:
> 시야 가장자리에 흐릿하게 떠 있던 줄들이 또렷해진다. 모노스페이스 폰트, 녹색 글자. 누구도 보지 못하는 화면이 내 안구 안쪽에서 펼쳐진다. 손가락이 무의식적으로 키보드 자세를 잡는다.

### 4-2. 빌드 룸 환각

- 빌드 룸은 물리적 공간이 아니다. 강해가 눈을 감고 의식을 집중하면, 시야 앞에 **거대한 IDE 창**이 뜬다. 자취방 벽이 사라지고 코드만 남는 환각.
- 편집 시: 손가락이 허공에서 키보드를 두드리듯 움직인다. 본인은 무의식. 옆에서 보면 "...뭐 하고 있어" 소리 들음.
- 컴파일 시: 진행 바가 시야 하단에 뜨고, 점진적으로 채워진다. 빌드 시간 1~10초.
- 컴파일 완료: SUCCESS는 짧은 종소리, ERROR는 둔탁한 부저음 (강해 머릿속에서만 들림).

### 4-3. 디버깅 시 코드 라인 빛나는 연출

- 다른 헌터의 스킬 코드를 분석할 때, 문제 있는 라인이 **노랑 → 빨강** 단계로 빛난다.
- 강해의 시선이 그 라인에 멈추면, 라인 옆에 **빨간 ⚠ 마커**가 뜸 (IDE의 경고 아이콘 같은).
- 한지유 절대영도 코드를 봤을 때: `// double user_radius_m = ?;` 라인이 **회색**으로 죽어 있고, **점선 박스**로 둘러싸여 있음. 비활성 표시.

### 4-4. 콤보 발동 시 시각 연출

- 3개 D급 스킬의 코드 라인이 각각 **파랑·노랑·초록**으로 빛나며, 콤보 트리거 변수(`var t = System.nanoTime()`)가 활성화되는 순간 **세 라인이 하나의 흰 빛으로 합쳐짐**.
- 강해 시야에 한순간 IDE 창이 폭발하듯 닫히고, 현실 시야에 폭발 이펙트가 동기화. ("박자" 어휘 회피 — AI 말습관 가드레일)
- 콤보 후 **컴파일 로그가 시야 우측 상단**에 작게 뜸. `[combo_chain_cooldown=12s]` — 강해는 그걸 보고 12초 동안은 다시 못 쓴다고 인지.

### 4-5. 시스템 함수 발견 시 연출

- 일반 코드는 **녹색/흰색 모노스페이스**.
- 시스템 함수(`harvest_intention`, `report_anomaly`)는 **자주색/보라**. 다른 색조라서 강해 눈에 즉시 들어옴.
- 보호 영역 `// [PROTECTED]`은 **회색 + 점선 박스 + 자물쇠 아이콘**. 시선을 댈 수는 있지만, 편집 커서가 들어가면 빨간 X.
- EP025 `harvest_intention` 첫 발견: 보호 영역 안쪽에서 자주색 한 줄이 **잠깐 깜빡이고 사라짐**. 강해가 다시 보려고 시선을 고정해도 보이지 않음. "...분명히 봤는데."
