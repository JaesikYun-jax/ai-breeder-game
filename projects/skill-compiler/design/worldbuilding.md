## 2. 상태창 시스템

### 2-1. 보급형 (Status Window v1.0) — 일반인

| 항목 | 내용 |
|---|---|
| 표시 정보 | 본인 스탯, 본인 스킬 |
| 스킬 표현 | 아이콘 + 설명문 |
| 부작용 | 없음 |

### 2-2. 안강해의 오리지널 (v0.1 베타)

| 항목 | 내용 |
|---|---|
| 표시 정보 | 본인 + 타인 스탯/스킬, 일부 감정 텍스트 |
| 스킬 표현 | **코드 블록 (조건문, 변수, 함수)** |
| 부작용 | 타인 마음 누수 (불수의) — 군중 속 두통, 강한 감정 시 의식 끊김 |

### 2-3. 2차 각성 = "스킬 컴파일러" (대각성 1년 후, EP001 직전)

| 단계 | 능력 |
|---|---|
| 읽기 | 본인/타인 스킬 코드 시각화 (기존 능력 확장) |
| 편집 | 본인 스킬만 수정 가능. 타인은 읽기 전용 |
| 컴파일 | 수정한 스킬을 사용 가능 형태로 빌드 |

## 3. 능력 제약 (밸런스 잠금)

1. **빌드 타임 제약**: 전투 중 수정 불가. 던전 입장 전 '스킬 빌드 룸'에서만. 한 번 들어가면 그 빌드로 끝
2. **스킬 슬롯 제한**: 동시 컴파일 가능 N개 (초반 3 → 성장하며 확장)
3. **스킬 등급 캡**: F~D급 스킬만 자유 수정. C급 이상은 "주석 처리된 보호 영역"이 있어 부분 수정만 가능
4. **버그 리스크**: 수정 스킬은 일정 확률로 런타임 에러(역소환·자해 데미지·침묵). 테스트→안정화 사이클 필요

## 6. 시스템의 진실 (1부 후반 ~ 떡밥)

- 상태창 = 인류 감정을 수확하는 장치 (의념 탈수기)
- 게이트·던전·레이드 = 농장 사이클: 절망 → 희망 → 환희 → 다시 절망
- 강한 감정일수록 양분 큼 → 사이다와 고난 정교 배치
- 강해는 베타 테스터였기에 코드를 직접 본 유일한 존재
- 0층(시스템 설계자) — 신? AI? 더 상위 차원? — 후반 공개
